# Глава 3. Программная реализация

## 3.1. Анализ и выбор СУБД

### 3.1.1. Обоснование выбора PostgreSQL

Для реализации базы данных выбрана **СУБД PostgreSQL** версии 14 и выше.

**Критерии выбора:**
- **Стоимость:** Бесплатная открытая лицензия PostgreSQL
- **Масштабируемость:** Поддержка больших объёмов данных и высоких нагрузок
- **Безопасность:** Встроенные механизмы аутентификации и разграничения доступа
- **Функциональность:** Поддержка представлений, триггеров, хранимых процедур
- **Надёжность:** ACID-совместимость, транзакционная целостность
- **Сообщество:** Активное сообщество, обширная документация

### 3.1.2. Характеристика PostgreSQL

PostgreSQL — объектно-реляционная система управления базами данных с открытым исходным кодом. Система поддерживает SQL стандарт ANSI/ISO и обладает расширенными возможностями.

**Основные возможности:**
- Поддержка сложных типов данных (JSON, массивы, диапазоны)
- Надёжные транзакции с многоверсионным управлением (MVCC)
- Расширяемая архитектура
- Полнотекстовый поиск
- Параллельные запросы

-- Таблица Успеваемость
CREATE TABLE Успеваемость (

## 3.3. Разработка представлений (Views)

### 3.3.1. Представление списка клиентов с договорами

```sql
CREATE VIEW КлиентыСДоговорами AS
SELECT 
    к.фамилия || ' ' || к.имя || ' ' || COALESCE(к.отчество, '') AS ФИО,
    к.телефон,
    к.email,
    д.номер_договора,
    д.дата_заключения,
    д.сумма AS сумма_договора,
    д.статус AS статус_договора
FROM Клиент к
LEFT JOIN Договор д ON к.id_клиента = д.id_клиента
ORDER BY к.фамилия, к.имя;
```

### 3.3.2. Представление расписания группы

```sql
CREATE VIEW РасписаниеГруппы AS
SELECT 
    г.название AS группа,
    к.название AS курс,
    п.фамилия || ' ' || п.имя AS преподаватель,
    з.номер_занятия,
    з.дата_занятия,
    з.время_начала,
    з.время_конца,
    з.аудитория,
    з.тема_занятия
FROM Занятие з
JOIN Группа г ON з.id_группы = г.id_группы
JOIN Курс к ON г.id_курса = к.id_курса
JOIN Преподаватель п ON г.id_преподавателя = п.id_преподавателя
ORDER BY з.дата_занятия, з.время_начала;
```

### 3.3.3. Представление успеваемости группы

```sql
CREATE VIEW УспеваемостьГруппы AS
SELECT 
    г.название AS группа,
    к.название AS курс,
    клиент.фамилия || ' ' || клиент.имя AS слушатель,
    з.номер_занятия,
    з.тема_занятия,
    у.оценка,
    у.домашнее_задание,
    у.проверено
FROM Успеваемость у
JOIN Занятие з ON у.id_занятия = з.id_занятия
JOIN Группа г ON з.id_группы = г.id_группы
JOIN Курс к ON г.id_курса = к.id_курса
JOIN Клиент клиент ON у.id_клиента = клиент.id_клиента
ORDER BY г.название, з.номер_занятия;
```

### 3.3.4. Представление задолженностей

```sql
CREATE VIEW Задолженности AS
SELECT 
    к.id_клиента,
    к.фамилия || ' ' || к.имя AS должник,
    д.номер_договора,
    д.сумма AS сумма_договора,
    COALESCE(SUM(оп.сумма), 0) AS оплачено,
    д.сумма - COALESCE(SUM(оп.сумма), 0) AS задолженность
FROM Клиент к
JOIN Договор д ON к.id_клиента = д.id_клиента
LEFT JOIN Оплата оп ON д.id_договора = оп.id_договора
WHERE д.статус = 'Действует'
GROUP BY к.id_клиента, к.фамилия, к.имя, д.id_договора, д.номер_договора, д.сумма
HAVING д.сумма - COALESCE(SUM(оп.сумма), 0) > 0
ORDER BY задолженность DESC;
```

### 3.3.5. Представление статистики курсов

```sql
CREATE VIEW СтатистикаКурсов AS
SELECT 
    к.id_курса,
    к.название,
    к.продолжительность_часов,
    к.стоимость,
    COUNT(DISTINCT г.id_группы) AS количество_групп,
    COUNT(DISTINCT д.id_клиента) AS количество_слушателей,
    COUNT(DISTINCT ce.id_сертификата) AS выдано_сертификатов
FROM Курс к
LEFT JOIN Группа г ON к.id_курса = г.id_курса
LEFT JOIN Договор д ON г.id_группы = д.id_группы
LEFT JOIN Сертификат ce ON к.id_курса = ce.id_курса
GROUP BY к.id_курса, к.название, к.продолжительность_часов, к.стоимость
ORDER BY количество_слушателей DESC;
```

## 3.4. Разработка форм

Для работы с базой данных разработаны следующие формы:

### 3.4.1. Форма «Клиенты»

Назначение: Добавление и редактирование информации о клиентах.

Поля формы:
- Фамилия (текстовое поле, обязательно)
- Имя (текстовое поле, обязательно)
- Отчество (текстовое поле)
- Дата рождения (поле выбора даты)
- Телефон (текстовое поле с маской)
- Email (текстовое поле с валидацией)
- Адрес (многострочное поле)

Кнопки: «Сохранить», «Отмена», «Поиск»

### 3.4.2. Форма «Запись на курс»

Назначение: Регистрация нового клиента и запись на курс.

Поля формы:
- Выбор клиента (из справочника или создать нового)
- Выбор курса (из справочника)
- Выбор группы (из существующих или создать новую)
- Дата заключения договора (автозаполнение текущей датой)
- Сумма договора (автоматический расчёт)

Кнопки: «Создать договор», «Отмена», «Печать договора»

### 3.4.3. Форма «Успеваемость»

Назначение: Ввод и проверка домашних заданий, выставление оценок.

Поля формы:
- Выбор группы
- Выбор занятия
- Список слушателей группы
- Оценка (выпадающий список 1-5)
- Домашнее задание (текстовое поле)
- Флаг «Проверено»

Кнопки: «Сохранить», «Отмена»

### 3.4.4. Форма «Посещаемость»

Назначение: Отметка присутствия/отсутствия на занятии.

Поля формы:
- Выбор группы
- Выбор даты занятия
- Таблица слушателей с флагами присутствия
- Причина отсутствия (текстовое поле, заполняется при отсутствии)

Кнопки: «Сохранить», «Отмена»

Назначение: Учёт финансовых поступлений.

Структура отчёта:
- Заголовок: «Финансовый отчёт за [месяц/год]»

## 3.6. Реализация ограничений и автоматизация (триггеры)

### 3.6.1. Триггер 1: Автоматический расчёт общей суммы оплат по договору

```sql
CREATE OR REPLACE FUNCTION fn_обновить_статус_оплаты()
RETURNS TRIGGER AS $$
BEGIN
    IF NEW.сумма > 0 THEN
        UPDATE Оплата
        SET статус = 'Оплачен'
        WHERE id_оплаты = NEW.id_оплаты;
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_обновить_статус_оплаты
AFTER INSERT ON Оплата
FOR EACH ROW EXECUTE FUNCTION fn_обновить_статус_оплаты();
```

### 3.6.2. Триггер 2: Проверка доступности мест в группе

```sql
CREATE OR REPLACE FUNCTION fn_проверить_место_в_группе()
RETURNS TRIGGER AS $$
DECLARE
    текущее_количество INTEGER;
    максимум_группы INTEGER;
BEGIN
    SELECT COUNT(*), г.максимальный_размер
    INTO текущее_количество, максимум_группы
    FROM Договор д
    JOIN Группа г ON д.id_группы = г.id_группы
    WHERE д.id_группы = NEW.id_группы AND д.статус = 'Действует'
    GROUP BY г.максимальный_размер;
    
    IF текущее_количество >= максимум_группы THEN
        RAISE EXCEPTION 'В группе нет свободных мест! Максимум: %', максимум_группы;
    END IF;
    
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_проверить_место_в_группе
BEFORE INSERT ON Договор
FOR EACH ROW EXECUTE FUNCTION fn_проверить_место_в_группе();
```

### 3.6.3. Триггер 3: Автоматическое закрытие группы при окончании дат

```sql
CREATE OR REPLACE FUNCTION fn_обновить_статус_группы()
RETURNS TRIGGER AS $$
BEGIN
    UPDATE Группа
    SET статус = 'Завершена'
    WHERE дата_конца < CURRENT_DATE AND статус = 'В процессе';
    
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_обновить_статус_группы
AFTER INSERT ON Занятие
FOR EACH ROW EXECUTE FUNCTION fn_обновить_статус_группы();
```

### 3.6.4. Триггер 4: Логирование изменений в таблице клиентов

```sql
CREATE TABLE ИсторияИзмененийКлиентов (
    id_изменения SERIAL PRIMARY KEY,
    id_клиента INTEGER,
    действие VARCHAR(10),
    фамилия_старая VARCHAR(50),
    фамилия_новая VARCHAR(50),
    имя_старое VARCHAR(50),
    имя_новое VARCHAR(50),
    дата_изменения TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE OR REPLACE FUNCTION fn_логировать_изменения_клиента()
RETURNS TRIGGER AS $$
BEGIN
    IF OLD IS NOT NULL THEN
        INSERT INTO ИсторияИзмененийКлиентов 
            (id_клиента, действие, фамилия_старая, фамилия_новая, имя_старое, имя_новое)
        VALUES (
            NEW.id_клиента,
            'UPDATE',
            OLD.фамилия,
            NEW.фамилия,
            OLD.имя,
            NEW.имя
        );
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_логировать_изменения_клиента
AFTER UPDATE ON Клиент
FOR EACH ROW EXECUTE FUNCTION fn_логировать_изменения_клиента();
```
