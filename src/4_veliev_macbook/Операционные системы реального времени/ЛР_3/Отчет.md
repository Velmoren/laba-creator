# Отчет по лабораторной работе №3
## Дисциплина: «Операционные системы реального времени»
**Тема: Управление доступом и администрирование субъектов системы**

### 1. Введение
Цель: Настройка многопользовательской среды и разграничение привилегий через механизм `sudo`. Стек: `/etc/passwd`, `chmod`, `chown`. Принцип: Наименьшие привилегии (Least Privilege).

### 2. Ход выполнения работы
1. Регистрация нового инженерного субъекта:
```bash
sudo adduser devops_junior
```
Ubuntu реализовала интерактивный процесс конфигурации UID и создания домашней директории.
![Интерактивный adduser. Конфигурация учетной записи инженера.](screenshot_1.png)

2. Настройка атрибутов безопасности для скрипта деплоя `prod_deploy.sh`:
- Создание группы: `sudo groupadd infrastructure_ops`
- Смена владельца: `sudo chown veliev:infrastructure_ops prod_deploy.sh`
- Ограничение прав: `chmod 750 prod_deploy.sh` (rwxr-x---)

![Аудит атрибутов безопасности через ls -l. Проверка группы sudo.](screenshot_2.png)

### 3. Технический анализ
Маска 750 гарантирует исключительный доступ на запись для владельца и право исполнения для группы эксплуатации. Использование `sudo` позволяет выполнять административные задачи без раскрытия пароля root, что соответствует стандартам безопасности ОСРВ. Проверка через `id` подтвердила вхождение пользователя в системную группу `sudo` и целевую группу `infrastructure_ops`.

### 4. Заключение
Политика доступа к инфраструктурным скриптам настроена. Изоляция данных в многопользовательской среде обеспечена.
