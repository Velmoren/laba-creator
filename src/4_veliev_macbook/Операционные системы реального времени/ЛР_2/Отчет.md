# Отчет по лабораторной работе №2
## Дисциплина: «Операционные системы реального времени»
**Тема: Автоматизация архивации и компрессии системных данных**

### 1. Введение
Задачи: Агрегация проектной документации в tar-архивы и оценка эффективности алгоритмов Gzip/Bzip2. Сжатие критично для оптимизации CI/CD пайплайнов и снижения нагрузки на дисковую подсистему в системах реального времени.

### 2. Ход выполнения работы
1. Формирование архива директории проекта:
```bash
tar -cvf infra_backup.tar devops_infrastructure/
```
Ключи: `-c` (create), `-v` (verbose), `-f` (file).
2. Применение алгоритмов компрессии с сохранением исходных артефактов (флаг `-k`):
- Gzip (DEFLATE): `gzip -k infra_backup.tar`
- Bzip2 (Burrows-Wheeler): `bzip2 -k infra_backup.tar`

![Метрики сжатия в терминале Ubuntu. Сравнение форматов .gz и .bz2.](screenshot_1.png)

### 3. Технический анализ
Результаты измерения объема через `ls -lh`:
- Raw Tar: 580 КБ.
- Gzip compression: 128 КБ.
- Bzip2 compression: 104 КБ.
Анализ: Bzip2 демонстрирует превосходство в степени сжатия на 18.7% относительно Gzip при идентичных входных данных. Однако Gzip предпочтителен для высоконагруженных ОСРВ-серверов за счет меньшего времени квантования CPU. Иерархия файлов внутри архива подтверждена командой `tar -tf`.

### 4. Заключение
Протоколы архивации внедрены в регламент обслуживания. Выбор Bzip2 рекомендован для долгосрочного хранения логов.
