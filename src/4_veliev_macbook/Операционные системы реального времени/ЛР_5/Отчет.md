# Отчет по лабораторной работе №5
## Дисциплина: «Операционные системы реального времени»
**Тема: Автоматизация инфраструктурных задач на языке Bash**

### 1. Введение
Цель: Разработка и отладка пакета утилит для автоматизации администрирования Ubuntu. Стек: bash 5.1, GNU coreutils. Принципы: модульность, типизация данных, обработка исключений. Использование shell-скриптов в ОСРВ минимизирует время отклика при выполнении типовых операций эксплуатации.

### 2. Ход выполнения работы
Реализован комплекс из 11 скриптов автоматизации, размещенных в директории `source/`.

#### 2.1 Спецификация пакета утилит
1. **ops_args.sh**: Валидация входных параметров CLI с контролем количества аргументов.
2. **ops_math.sh**: Арифметический модуль для расчета формулы `D=(A*2 + B/3)*C` через встроенный интерпретатор.
3. **ops_home_export.sh**: Экспорт иерархии домашней директории в текстовый отчет с метаданными.
4. **ops_verify_user.sh**: Сверка UID активной сессии с идентификатором исследователя.
5. **ops_mime_check.sh**: Определение типа содержимого системных объектов через `file`.
6. **ops_find_fresh.sh**: Поиск артефактов, модифицированных в течение последних 24 часов.
7. **ops_link_resolve.sh**: Трассировка символических ссылок до конечной цели в VFS.
8. **ops_regex_count.sh**: Статистическая обработка вхождений паттернов в неструктурированные данные.
9. **ops_inode_map.sh**: Идентификация жестких ссылок по уникальному дескриптору инода.
10. **ops_quota_check.sh**: Расчет распределения дисковых квот между пользователем и системой.
11. **ops_path_validator.sh**: Аудит безопасности и доступности директорий в переменной `$PATH`.

#### 2.2 Верификация
Тестирование модуля `ops_math.sh` для подтверждения точности вычислений.

### 3. Технический анализ
Сценарии используют расширенный синтаксис `[[ ... ]]` для повышения надежности логических проверок. Использование арифметики `$((...))` обеспечивает выполнение операций в пространстве процесса bash без порождения дочерних процессов, что оптимизирует производительность ОСРВ. Обеспечена корректная обработка путей с пробелами через строгое квотирование переменных. Скрипт `ops_path_validator.sh` успешно прошел тест на разбор путей с разделителем `:`.

### 4. Заключение
Пакет утилит готов к интеграции в систему автоматизации. Навыки разработки bash-скриптов подтверждены.
