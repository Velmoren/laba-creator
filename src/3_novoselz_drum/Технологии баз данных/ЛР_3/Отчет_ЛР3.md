# Технический дневник ЛР №3: Как я проектировал БД «Галерея» в Postgres

## 1. Введение
Сегодня в моем рабочем пространстве — задача по проектированию реальной базы данных для арт-галереи. Это уже не просто учебные таблицы студентов, тут всё серьезно: художники, их бессмертные работы и транзакции. Главная цель — разобраться с DDL и понять, чем `nextval` лучше автоинкремента в других СУБД.

## 2. Реализация
Я начал с создания последовательностей. В Postgres это крутой инструмент, который живет отдельно от таблицы.

```sql
CREATE SEQUENCE global_art_seq START 1;
CREATE SEQUENCE customer_seq START 1000;
```

Затем описал таблицу художников. В этот раз PK задал сразу при создании — это короче и понятнее для простых таблиц.

```sql
CREATE TABLE ARTIST (
    ArtistID int PRIMARY KEY DEFAULT nextval('global_art_seq'),
    Name text NOT NULL,
    Nationality varchar(50)
);
```

А вот для таблицы `CUSTOMER` решил пойти другим путем (второй способ из методички) — сначала создал структуру, а потом через `ALTER TABLE` навесил первичный ключ. Это удобно, если забыл сделать это сразу или если ключ сложный.

```sql
CREATE TABLE CUSTOMER (
    CustomerID int DEFAULT nextval('customer_seq'),
    Name text NOT NULL
);
ALTER TABLE CUSTOMER ADD CONSTRAINT pk_cust_id PRIMARY KEY (CustomerID);
```
