# Технический дневник ЛР №4: Защита данных и ускорение запросов

## 1. Что сегодня делаем
База без связей — это просто набор Excel-файлов. Сегодня я связываю всё в единую экосистему и настраиваю каскадные удаления. Главное — не допустить «висячих» картин, если художник будет удален из базы.

## 2. Ход работы
Применил `ON DELETE CASCADE`. Теперь, если мы уберем художника, Postgres сам почистит его работы. Это и есть ссылочная целостность.

```sql
ALTER TABLE WORK ADD CONSTRAINT fk_artist_link 
FOREIGN KEY (ArtistID) REFERENCES ARTIST(ArtistID) 
ON DELETE CASCADE;
```

Для ускорения поиска по базе (а в галерее сотни имен) я добавил индексы. Это как оглавление в книге.

```sql
CREATE INDEX idx_search_artist ON ARTIST(Name);
```
