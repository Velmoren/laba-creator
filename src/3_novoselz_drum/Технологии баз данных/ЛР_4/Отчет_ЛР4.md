## 1. ЦЕЛОСТНОСТЬ ЧЕРЕЗ CASCADE
Чтобы база не превратилась в свалку после удаления художника, я настроил FK_WORK_ARTIST с опцией ON DELETE CASCADE. Теперь при удалении записи в ARTIST, Postgres сам вычистит все связанные картины в таблице WORK. Это фундаментальное требование реляционной модели.

```sql
ALTER TABLE WORK ADD CONSTRAINT fk_work_artist 
FOREIGN KEY (ArtistID) REFERENCES ARTIST(ArtistID) 
ON DELETE CASCADE;
```

## 2. ОПТИМИЗАЦИЯ И ИНДЕКСЫ
Я создал B-tree индексы для текстовых полей. Теперь поиск по имени художника ('Aivazovsky') будет мгновенным, так как СУБД будет использовать сбалансированное дерево вместо полного сканирования таблицы.

```sql
CREATE INDEX idx_art_name ON ARTIST(Name);
```

## 3. КОНТРОЛЬНОЕ ЗАДАНИЕ
Я расширил таблицу CUSTOMER, добавив поле SecondName. Теперь мы можем хранить фамилии отдельно от имен, что упрощает сортировку.

```sql
ALTER TABLE CUSTOMER ADD COLUMN SecondName varchar(255);
```
