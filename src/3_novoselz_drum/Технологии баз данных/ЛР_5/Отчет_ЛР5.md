## 1. ВСТУПЛЕНИЕ И МОТИВАЦИЯ
Привет! Сегодня в моем дневнике — одна из самых интересных тем в мире БД. Хватит слать сотни запросов из приложения, пора заставить базу саму думать и обрабатывать данные на месте. Мы переходим к PL/pgSQL (Procedural Language/PostgreSQL). Почему это круто? Это экономит ресурсы сети, убирает лишние задержки и позволяет упаковать сложную логику прямо в ядро БД. Моя цель сегодня — создать процедуру автоматического связывания клиентов с художниками.

## 2. АНАЛИЗ ИНСТРУМЕНТАРИЯ: КУРСОРЫ
Перед тем как писать код, я разобрался, зачем нужен CURSOR. Представьте, что запрос возвращает 1000 строк. Если загрузить их все в память разом — всё упадет. Курсор позволяет нам идти по результату «шагами», строка за строкой. Это как закладка в огромной книге.

**Этапы жизни моего курсора:**
1. **DECLARE:** Объявляю курсор, привязывая его к SELECT-запросу по национальностям.
2. **OPEN:** Заставляю базу выполнить запрос и подготовить результат.
3. **FETCH:** Вытягиваю текущую строку для обработки.
4. **CLOSE:** Освобождаю ресурсы после работы.

## 3. РЕАЛИЗАЦИЯ ПРОЦЕДУРЫ CUSTOMERINSERT
Я написал процедуру, которая делает две вещи: проверяет, нет ли уже такого клиента в базе (чтобы не плодить дубли), и если всё чисто — создает его и привязывает ко всем художникам его любимой нации.

```sql
-- Процедура для Новосельцева (Вар 18)
CREATE OR REPLACE PROCEDURE CustomerInsert (
    p_name varchar, p_area varchar, p_phone varchar, p_nation varchar
) AS $$
DECLARE
    -- Объявляю курсор для поиска художников
    artist_cur CURSOR FOR 
        SELECT ArtistID FROM ARTIST WHERE Nationality = p_nation;
    artist_rec RECORD;
    v_exists int;
BEGIN
    -- Шаг 1: Проверка на вшивость (дубликаты)
    SELECT count(*) INTO v_exists FROM CUSTOMER 
    WHERE Name = p_name AND PhoneNumber = p_phone;

    IF v_exists > 0 THEN
        RAISE NOTICE 'Новосельцев, клиент % уже в системе. Отмена.', p_name;
        RETURN;
    END IF;

    -- Шаг 2: Вставляем свежую кровь
    INSERT INTO CUSTOMER(Name, AreaCode, PhoneNumber)
    VALUES (p_name, p_area, p_phone);

    -- Шаг 3: Магия курсора - связываем интересы
    -- Использую цикл FOR, который сам откроет и закроет курсор
    FOR artist_rec IN artist_cur LOOP
        INSERT INTO CUSTOMER_ARTIST_INT(ArtistID, CustomerID)
        VALUES (artist_rec.ArtistID, currval('CustID_Seq'));
    END LOOP;

    RAISE NOTICE 'Клиент % успешно добавлен и связан с художниками: %', p_name, p_nation;
END;
$$ LANGUAGE plpgsql;
```
