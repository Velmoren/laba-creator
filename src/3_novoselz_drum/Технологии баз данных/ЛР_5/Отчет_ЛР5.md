## 1. ПОЧЕМУ PL/PGSQL?
Я решил перенести логику связывания клиента с художниками прямо в базу. Это экономит трафик и гарантирует, что бизнес-правила будут соблюдаться независимо от того, какое приложение подключается к БД.

## 2. РЕАЛИЗАЦИЯ CUSTOMERINSERT
Я написал процедуру, использующую переменную-курсор. Это позволяет перебирать художников по национальности один за другим. В цикле FOR я вставляю связи в таблицу интересов. 

```sql
CREATE OR REPLACE PROCEDURE CustomerInsert (
    p_name varchar, p_nation varchar
) AS $$
DECLARE
    art_rec RECORD;
    art_cur CURSOR FOR SELECT ArtistID FROM ARTIST WHERE Nationality = p_nation;
BEGIN
    INSERT INTO CUSTOMER(Name) VALUES (p_name);
    
    FOR art_rec IN art_cur LOOP
        INSERT INTO CUSTOMER_ARTIST_INT(ArtistID, CustomerID) 
        VALUES (art_rec.ArtistID, currval('CustID_Seq'));
    END LOOP;
    
    RAISE NOTICE 'Новосельцев, клиент % добавлен.', p_name;
END; $$ LANGUAGE plpgsql;
```
