# ОТЧЕТ ПО РАБОТЕ №6: УПРАВЛЕНИЕ ТРАНЗАКЦИЯМИ В ХРАНИМЫХ ПРОЦЕДУРАХ

## ВВЕДЕНИЕ
Целостность данных в многопользовательских системах обеспечивается механизмом транзакций. В данной работе реализуется процедура оформления покупки картины, требующая строгого соблюдения условий атомарности.

## 1. ЛОГИКА ПРОЦЕДУРЫ NEWCUSTOMERWITHTRANSACTION
Процедура обрабатывает сложный процесс: добавление клиента и регистрацию продажи только при условии наличия свободной копии картины в таблице TRANSACTION.

```sql
-- Фрагмент логики с ROLLBACK
IF v_rowcount = 0 THEN
    ROLLBACK; -- Отмена всех действий, если картина не найдена
    RAISE EXCEPTION 'Нет свободных копий данной работы';
ELSIF v_rowcount > 1 THEN
    ROLLBACK; -- Отмена, если данные противоречивы
    RAISE EXCEPTION 'Обнаружено дублирование транзакций';
ELSE
    -- Успешное выполнение
    UPDATE TRANSACTION SET CustomerID = v_cid, PurchaseDate = current_date ...
    COMMIT;
END IF;
```

## ЗАКЛЮЧЕНИЕ
Реализованный механизм гарантирует, что клиент не будет добавлен в базу, если сделка по картине не может быть совершена. Это классический пример соблюдения принципов ACID.
