# ОТЧЕТ ПО РАБОТЕ №8: ТРИГГЕРЫ ПРЕДСТАВЛЕНИЙ И ОБРАБОТКА ИСКЛЮЧИТЕЛЬНЫХ СИТУАЦИЙ

## ВВЕДЕНИЕ
Заключительная работа посвящена продвинутым методам работы с представлениями (View) и обеспечению отказоустойчивости серверного кода.

## 1. ТРИГГЕР INSTEAD OF
Поскольку представление `CustomerPurchases` является объединением нескольких таблиц, оно недоступно для прямого обновления. Реализован триггер, перехватывающий попытку изменения и направляющий её в базовую таблицу `WORK`.

```sql
CREATE TRIGGER trg_update_view
INSTEAD OF UPDATE ON CustomerPurchases
FOR EACH ROW EXECUTE FUNCTION fn_update_work_title();
```

## 2. ОБРАБОТКА ОШИБОК
Для вывода детальной информации о сбоях использован блок `EXCEPTION`.

```sql
EXCEPTION WHEN others THEN
    GET STACKED DIAGNOSTICS v_msg = MESSAGE_TEXT;
    RAISE NOTICE 'Произошла ошибка: %', v_msg;
```
