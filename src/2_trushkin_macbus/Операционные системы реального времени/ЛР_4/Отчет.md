# Отчет по лабораторной работе №4
## Дисциплина: «Операционные системы реального времени»
**Тема: Инструментарий потоковой обработки текстовых данных и применение регулярных выражений в Ubuntu Linux**

### 1. Теоретическое введение
Фундаментальная парадигма операционных систем семейства UNIX («все есть файл») подразумевает, что подавляющее большинство системных конфигураций, журналов состояния и баз данных субъектов представлены в текстовом формате. Для эффективного анализа таких массивов данных в Ubuntu применяется набор потоковых фильтров GNU Coreutils. Центральное место в данном инструментарии занимает утилита `grep` (Global Regular Expression Print), предназначенная для поиска и фильтрации строк, соответствующих заданным регулярным выражениям. Эффективность администрирования ОСРВ кратно возрастает при использовании механизма конвейеров (pipes, символ `|`), позволяющего объединять атомарные утилиты в сложные аналитические пайплайны, минимизируя накладные расходы на операции ввода-вывода.

### 2. Ход выполнения работы
В ходе экспериментального исследования была проведена серия аналитических выборок из имитированных системных логов.
1. Локализация критических аномалий в журнале `research_syslog.log`:
```bash
grep -E "CRITICAL|FAILURE" research_syslog.log | tail -n 5
```
Использование флага `-E` позволило применить расширенные регулярные выражения для объединения нескольких паттернов поиска.
![Результаты фильтрации системных журналов на предмет критических ошибок](screenshot_1.png)

2. Статистический анализ попыток неавторизованного доступа:
```bash
grep "Invalid user" auth_attempts.txt | cut -d' ' -f8 | sort | uniq -c
```
3. Идентификация пользователей с интерактивной оболочкой bash в системной базе данных:
```bash
cat /etc/passwd | grep "/bin/bash" | cut -d: -f1
```

### 3. Технический анализ результатов
Применение команды `grep` в связке с `cut` и `uniq` позволило оперативно трансформировать неструктурированные текстовые логи в структурированный отчет о состоянии системы. Эмпирически подтверждено, что конвейерная обработка в Ubuntu выполняется в оперативной памяти без создания временных файлов на диске, что критично для производительности ОСРВ. В терминале Ubuntu результаты поиска по умолчанию подсвечиваются цветом (кодировка ANSI), что значительно облегчает визуальную верификацию данных исследователем при анализе больших дампов логов.

### 4. Заключение
Освоение инструментов потоковой фильтрации является необходимым условием для квалифицированного мониторинга и диагностики ОСРВ Ubuntu. Сочетание утилиты `grep` с регулярными выражениями предоставляет мощный математический аппарат для обработки системной информации в режиме реального времени.
