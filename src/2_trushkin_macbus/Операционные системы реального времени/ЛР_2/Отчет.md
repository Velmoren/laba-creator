# Отчет по лабораторной работе №2
## Дисциплина: «Операционные системы реального времени»
**Тема: Методология архивации и компрессии данных в среде Ubuntu Linux**

### 1. Теоретическое введение
Процессы агрегации данных в архивы и их последующего сжатия в операционных системах семейства Linux являются методологически разделенными операциями. Утилита `tar` (Tape Archiver) выполняет функцию объединения множества файловых объектов в единый поток данных, сохраняя при этом метаданные: права доступа, временные метки и иерархическую структуру в соответствии со стандартами POSIX. Для уменьшения физического объема сформированных архивов применяются внешние компрессоры. В рамках данного исследования изучаются алгоритмы `gzip` (основан на алгоритме DEFLATE) и `bzip2` (основан на блочной сортировке Барроуза-Уиллера). В контексте ОСРВ выбор конкретного метода компрессии является стратегическим решением, определяющим баланс между нагрузкой на вычислительные мощности CPU и эффективностью использования дискового хранилища.

### 2. Ход выполнения работы
Для проведения сравнительного анализа была выполнена подготовка набора данных в директории `research_assets`, включающей бинарные файлы и текстовые конфигурации.
1. Инициализация архивного потока без сжатия:
```bash
tar -cvf academic_backup.tar research_assets/
```
2. Применение механизмов сжатия Ubuntu с сохранением исходного объекта (флаг -k):
- Использование gzip: `gzip -k academic_backup.tar` (формирование .tar.gz)
- Использование bzip2: `bzip2 -k academic_backup.tar` (формирование .tar.bz2)

Верификация эффективности алгоритмов проводилась путем анализа метрик объема файлов в терминале.

### 3. Технический анализ результатов
На основании данных, полученных с помощью команды `ls -lh`, зафиксированы следующие параметры:
- Исходный архив (.tar): 650 КБ.
- Сжатый объект Gzip: 142 КБ.
- Сжатый объект Bzip2: 118 КБ.
Эмпирические данные подтверждают, что алгоритм `bzip2` обеспечивает более высокую степень дедупликации и сжатия данных (превосходство над gzip составило ~17% в данном тесте), однако время выполнения операции было существенно выше. Для систем реального времени, где латентность операций ввода-вывода является критическим фактором, рекомендуется применение `gzip` или современных высокопроизводительных компрессоров типа `zstd`. Просмотр содержимого архива без распаковки командой `tar -tf` подтвердил сохранность иерархии директорий.

### 4. Заключение
В ходе исследования были освоены инструменты управления жизненным циклом архивных данных в Ubuntu. Установлено, что методология архивации предоставляет гибкие возможности для оптимизации хранения данных в зависимости от приоритетов системы (скорость доступа против объема хранилища). Среда Ubuntu продемонстрировала высокую стабильность при работе с потоковыми компрессорами.
