# Отчет по лабораторной работе №3
## Дисциплина: «Операционные системы реального времени»
**Тема: Администрирование субъектов системы и разграничение прав доступа в Ubuntu Linux**

### 1. Теоретическое введение
Обеспечение информационной безопасности и изоляции вычислительных процессов в Ubuntu Linux базируется на классической модели дискреционного управления доступом (DAC), соответствующей стандартам POSIX. В данной архитектурной модели каждый системный объект (файл, каталог, сокет) ассоциирован с уникальным идентификатором владельца (UID) и группы (GID). Права доступа детерминируются набором атрибутов: r (чтение), w (запись) и x (исполнение). Ключевым инструментом администрирования в Ubuntu является механизм `sudo` (SuperUser DO), позволяющий делегировать привилегии суперпользователя авторизованным субъектам на основе конфигурационного файла `/etc/sudoers`. Процесс управления пользователями подразумевает взаимодействие с системными базами данных `/etc/passwd` и `/etc/shadow`, где хранятся метаданные и хешированные аутентификационные данные соответственно.

### 2. Ход выполнения работы
В рамках экспериментального исследования был инициирован процесс регистрации в системе нового вспомогательного субъекта:
```bash
sudo adduser research_assistant
```
Процесс сопровождался интерактивным запросом метаданных (Full Name, Room Number) и автоматическим формированием домашней директории в соответствии с шаблоном `/etc/skel`.
![Интерактивный процесс создания учетной записи пользователя в Ubuntu](screenshot_1.png)

Далее были проведены манипуляции с атрибутами безопасности для исследовательского журнала `security_audit.log`:
1. Формирование новой системной группы: `sudo groupadd research_grp`
2. Смена владельца и группы объекта: `sudo chown trushkin:research_grp security_audit.log`
3. Установка ограничительной маски прав доступа: `chmod 750 security_audit.log` (rwxr-x---)

![Эмпирическая верификация установленных масок доступа и принадлежности файла](screenshot_2.png)

### 3. Технический анализ результатов
Аналитическая оценка вывода команды `ls -l` подтвердила корректность применения установленной политики безопасности. Маска 750 гарантирует полный контроль над объектом для владельца (`trushkin`), возможность чтения и перехода для членов группы `research_grp` и абсолютный запрет на любые операции для неавторизованных субъектов (others). Данный подход реализует принцип наименьших привилегий (Principle of Least Privilege), минимизируя риски компрометации критических данных ОСРВ. Проверка через утилиту `id` подтвердила успешное включение пользователя в дополнительные системные группы.

### 4. Заключение
В ходе выполнения работы были освоены фундаментальные методы разграничения прав доступа в Ubuntu. Полученные навыки позволяют эффективно проектировать многопользовательские среды с заданным уровнем изоляции данных, что является критическим требованием для современных операционных систем реального времени.
