# Отчет по лабораторной работе №7
## Дисциплина: «Операционные системы реального времени»
**Тема: Анализ подсистемы хранения данных и механизмов монтирования в Ubuntu Linux**

### 1. Теоретическое введение
Взаимодействие с устройствами хранения в Ubuntu Linux основано на концепции блочных устройств, представленных в виртуальной файловой системе в директории `/dev/`. Процесс интеграции файловой системы внешнего носителя в общее дерево каталогов называется монтированием. Утилита `mount` обеспечивает связывание физического раздела диска с точкой монтирования (mount point) на уровне VFS, а механизмы ядра гарантируют прозрачный доступ к данным. Для мониторинга и анализа утилизации дискового пространства применяются системные утилиты `df` (disk free) и `du` (disk usage), позволяющие оценивать состояние разделов и объемы конкретных директорий в удобном для исследователя человекочитаемом формате.

### 2. Ход выполнения работы
Для исследования дисковой подсистемы была проведена серия диагностических операций.
1. Верификация иерархии блочных устройств и разделов:
```bash
lsblk
```
2. Анализ свободного пространства на активных файловых системах:
```bash
df -h | grep "^/dev/"
```
![Статистический анализ утилизации разделов и структуры блочных устройств](screenshot_1.png)

3. Монтирование исследовательского тома данных:
```bash
sudo mkdir -p /mnt/research_volume
sudo mount -t vfat /dev/sdb1 /mnt/research_volume
ls -l /mnt/research_volume
```
4. Определение объема директории с результатами практикума:
```bash
du -sh trushkin_research/
```

### 3. Технический анализ результатов
На основании полученных метрик установлено, что системный раздел (`/`) загружен на 42%, что является оптимальным показателем для обеспечения стабильности ОСРВ. Использование флага `-h` (human-readable) позволило зафиксировать объем данных в гигабайтах и мегабайтах без необходимости ручного пересчета блоков. Команда `lsblk` наглядно продемонстрировала топологическую зависимость между физическими накопителями и логическими разделами. В ходе выполнения монтирования было подтверждено, что права доступа к точке монтирования корректно наследуют параметры файловой системы целевого носителя. Результаты аудита сохранены в файле `storage_audit.txt`.

### 4. Заключение
В ходе работы были освоены фундаментальные методы управления хранилищами данных в Ubuntu. Понимание принципов монтирования и анализа дисковых ресурсов является критическим навыком для обеспечения масштабируемости и надежности современных операционных систем реального времени.
