# Лабораторная работа №6 «Управление процессами и автоматизация заданий во FreeBSD»

**Дисциплина:** Операционные системы реального времени
**Кафедра:** Интеллектуальные системы в управлении и автоматизации

## Цель работы
Изучение средств для мониторинга производительности, управления процессами и автоматизации заданий в ОС FreeBSD.

## Теоретические сведения
Во FreeBSD для управления процессами используются утилиты `ps` и `top`.
- `top` — динамический мониторинг процессов.
- `ps -aux` — подробный список всех процессов.
- `renice` — изменение приоритета процесса.
- `&` — запуск процесса в фоновом режиме.
- `cron` — планировщик периодических заданий.
- `crontab -e` — редактирование таблицы заданий пользователя.

## Задание
1. Посмотреть работу утилит `top` и `ps`.
2. Посмотреть приоритет своего процесса и уменьшить скорость его выполнения за счет повышения номера приоритета (`renice`).
3. Создать программу на Си с бесконечным циклом, откомпилировать её и запустить в фоновом режиме.
4. Прервать выполнение этой программы (`kill`).
5. Создать каталог в своём домашнем каталоге.
6. Создать сценарий shell для архивации другого непустого каталога:
   - Проверка непустой ли каталог.
   - Если непустой, то все файлы архивируются и сжимаются с именем, включающим дату и время.
   - Сжатый архив помещается в другой каталог.
7. Сделать сценарий исполняемым и добавить его в `cron` для выполнения каждые две минуты.
8. Проверить работу и удалить задание из `cron`.

## Порядок выполнения
1. Мониторинг: запустите `top` и посмотрите столбец `PRI`.
2. Программа на Си (`infinite.c`):
   ```c
   int main() { while(1); return 0; }
   ```
3. Компиляция и запуск: `gcc infinite.c -o infinite && ./infinite &`.
4. Остановка: `kill PID`.
5. Скрипт архивации (`archive.sh`):
   ```bash
   #!/bin/sh
   if [ "$(ls -A $1)" ]; then
     tar -czf $2/backup_$(date +%Y%m%d_%H%M).tar.gz $1/*
   fi
   ```
6. Настройка `cron`: `crontab -e` -> `*/2 * * * * /path/to/archive.sh /from /to`.
