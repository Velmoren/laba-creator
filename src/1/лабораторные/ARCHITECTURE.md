# Архитектура методических указаний по дисциплине «ОСРВ» (FreeBSD)

## 1. Общие положения
Каждая лабораторная работа должна быть оформлена в соответствии с ГОСТ МТУСИ и представлять собой законченное методическое руководство для студента.

## 2. Структура файла (Markdown)
Файл каждой ЛР должен содержать следующие разделы (в указанном порядке):

1.  **Заголовок первого уровня (#)**: Лабораторная работа №X «Название».
2.  **Цель работы**: Краткая формулировка навыков и знаний, получаемых в ходе ЛР.
3.  **Теоретические сведения**: Описание используемых команд, системных вызовов или концепций FreeBSD.
4.  **Задание**: Перечень задач, которые должен выполнить студент. Должно включать разделение по вариантам (минимум 10 вариантов).
5.  **Порядок выполнения**: Пошаговый алгоритм действий с примерами синтаксиса.
6.  **Контрольные вопросы**: 5-10 вопросов для проверки знаний на защите.
7.  **Требования к отчету**: Список того, что должно быть в отчете (скриншоты, листинги, ответы на вопросы).

## 3. Стилизация (для генератора)
- **Код**: Оформляется блоками ` ```bash ... ``` `.
- **Варианты**: Оформляются в виде таблиц Markdown.
- **Рисунки/Схемы**: Если требуются диаграммы, использовать Mermaid (` ```mermaid ... ``` `).
- **Выделение**: Ключевые термины — жирным, команды — моноширинным шрифтом.

## 4. Механизм генерации
Для получения `.docx` используется модифицированный скрипт `scripts/generate_labs.js`, который:
- Подставляет актуальный учебный год.
- Формирует титульный лист для методических указаний.
- Сохраняет каждую работу в `docs/labs/LR_X.docx`.
