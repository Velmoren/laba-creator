# Навык «Инфраструктура создания ПО» (ОС Альт)

## 1. Технический контекст
Работы выполняются в среде отечественной операционной системы **«Альт»** (платформа p10/p11).
- **Архитектура:** x86_64 (вместо e2k из пособия).
- **Инструментарий:** `apt-get`, `rpm`, `rpmbuild`, `hasher` (hsh), `gear`.
- **Базовый пакет:** `HelloUniverse`.

## 2. Спецификация лабораторных работ

### ЛР №1: Базовые инструменты RPM (Пункт 7.1 пособия)
- **Цель:** Освоение команд управления установленными пакетами.
- **Действия:** 
  1. Установка: `sudo apt-get install HelloUniverse`.
  2. Запуск: выполнение команды `HelloUniverse` (вывод текстового приветствия).
  3. Удаление: `sudo apt-get remove HelloUniverse`.
- **Скриншот:** Один общий скриншот терминала, показывающий последовательно все три команды и их вывод.

### ЛР №2: Сборка пакетов в среде Hasher (Пункт 7.2 пособия)
- **Цель:** Изучение методов сборки ПО в изолированном окружении (chroot).
- **Сценарии (3 скриншота):**
  1. **Пример 2.1 (Ручная сборка):** Результат команды `rpmbuild -bb` внутри `hsh-shell`.
  2. **Пример 2.2 (Сборка из .src.rpm):** Использование `hsh --verbose ~/RPM/SRPMS/HelloUniverse-*.src.rpm`.
  3. **Пример 2.3 (Сборка из tar-архива):** Использование `hsh --verbose ./pkg.tar`.
- **Особенности:** В путях использовать архитектуру `x86_64`.

## 3. Требования к контенту и промптам
- **Объем отчета:** 8000-10000 знаков. Подробное описание каждого ключа команд (например, что значат `-bb` или `--initroot-only`).
- **Синхронизация (Zero Contradiction):** 
  - Имя пользователя в терминале (`[user@altlinux]`) должно совпадать с `username` из `student_settings.json`.
  - Версии пакетов и метки времени в промпте должны соответствовать тексту отчета.
- **Детализация промптов:** 2500+ символов. Описание шрифтов (например, "Terminus" или "Monospace"), цветов приглашения Bash, ASCII-арта ОС Альт при входе.
- **Минимизация шума:** Только окно терминала на чистом фоне.

## 4. Протокол валидации
1. **Файловая проверка:** Наличие отчета (`.md`), рабочего скрипта сценария (`.sh`) в `source/` и детальных промптов.
2. **Анти-флуд (КРИТИЧНО):** Запрещено искусственное увеличение объема текста путем дублирования строк, абзацев или блоков «Глубокого анализа». Весь контент должен быть уникальным и нести техническую ценность.
3. **Промпт-аудит:** Проверка объема (>2500 симв) и отсутствия лишних элементов GUI. Запрещено копирование одних и тех же строк описания деталей (Detail: ...).
4. **Сверка данных:** Построчное сопоставление команд из отчета с командами, описанными в промпте. Любое расхождение в путях или версиях — критическая ошибка.
