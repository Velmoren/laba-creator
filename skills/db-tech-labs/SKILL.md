# Генератор лабораторных работ «Технологии баз данных»

Этот навык предназначен для генерации уникальных пакетов лабораторных работ (Отчеты + SQL-скрипты + промпты для скриншотов) по дисциплине «Технологии баз данных».

## 1. Входные параметры

При вызове навыка пользователь задает следующие переменные:
1. **Операционная система студента:** (Windows, macOS или Linux). Это влияет на текст промптов для скриншотов (например, `cmd.exe` vs `macOS terminal` vs `GNOME Terminal`).
2. **Фон рабочего стола:** (оттенок цвета или описание абстрактного рисунка). Это добавляется в промпты, но с жестким ограничением: **никаких элементов рабочего стола (иконок, панелей задач) кроме самого окна терминала/программы быть не должно.**
3. **Вариант оформления / стиль кода:** Стилистика написания SQL (case, отступы), стиль отчета (Markdown-разметка, синонимы), имена таблиц и тестовые данные.

## 2. Алгоритм работы

1. **Определение варианта:** Используйте уникальный стиль таблиц, данных и текста, чтобы отчеты разных студентов не пересекались. Вы можете комбинировать разные стили Markdown (например, использование `#` для заголовков против HTML-тегов `<div align="center">`).
2. **Генерация отчета:** Создайте Markdown-файл отчета (`Отчет_ЛРX.md`). Структура отчета должна включать: Введение/Цель, Ход работы (содержащий SQL-запросы), Выводы. *Контрольные вопросы и титульные листы не добавлять, если не указано иное.*
3. **Генерация промптов для скриншотов:** Сгенерируйте промпты для ИИ (на английском языке) для создания реалистичных скриншотов терминала и GUI (PgAdmin).
   - Внедрите входные переменные **OS** и **Фон рабочего стола**.
   - Добавьте строгое ограничение: `The image should look like a direct screen capture showing only the application window (terminal or PgAdmin) over a slightly blurred background of [ФОН]. No desktop icons, no taskbars, no extra OS UI elements.`
4. **Сохранение результатов:** Сохраните сгенерированные файлы `.md` и `.txt` (промпты), а также `.sql` (скрипты и дампы) в целевую директорию студента.

## 3. Примеры (Референсы)

Для обеспечения качества генерации ориентируйтесь на примеры успешно выполненных лабораторных работ для двух разных студентов, которые находятся в папке `references/`.

- **Студент 1 (Вариант 1):** Находится в `references/student1/`.
  - Отличается использованием таблиц `courses`, `students`, `exams`.
  - Промпты для macOS и Linux.
  - Форматирование отчета простое (`#`, `##`, `###`).
- **Студент 2 (Вариант 2):** Находится в `references/student2/`.
  - Отличается использованием таблиц `subject`, `scholar`, `testing`.
  - Промпты для Windows cmd.exe и macOS.
  - Более формальный стиль описания действий.

Изучите эти файлы при необходимости создать новый, третий или четвертый уникальный вариант:
- Читайте `references/student1/Отчет_ЛР1.md` для понимания структуры первой лабораторной работы.
- Читайте `references/student2/Report_Lab2.md` для понимания глубины проработки транзакций и запросов второй лабораторной работы.
- Изучите `prompt_screenshot...txt` для понимания того, как детально и технически грамотно должен быть описан терминальный интерфейс для генератора изображений.
