# Навык «Базы данных»

Этот навык используется для генерации уникальных пакетов лабораторных работ по PostgreSQL. Каждая работа генерируется как отдельный объект.

## 1. Общие требования к генерации
1. **Source Folder:** Все SQL-скрипты, дампы и промпты должны находиться в подпапке `source/`.
2. **Уникальность:** Данные в таблицах (имена клиентов, названия картин, цены) должны быть уникальными для каждого студента.
3. **Объем отчета:** Каждый отчет должен быть не менее 3000-4000 знаков с подробным разбором кода.

---

## 2. Реестр лабораторных работ (1-8)

### ЛР №1: Установка и настройка СУБД
- **Задачи:** Установка PostgreSQL, создание БД, ролей и прав доступа в psql и PgAdmin.

### ЛР №2: Базовый SQL (DDL, DML, Транзакции)
- **Задачи:** Таблицы `courses`/`students`. Сложные выборки (JOIN, GROUP BY). Работа с `BEGIN/COMMIT`.

### ЛР №3: Проектирование схемы БД «View Ridge»
- **Область:** Создание таблиц `ARTIST`, `WORK`, `CUSTOMER`, `CUSTOMER_ARTIST_INT`.
- **Задачи:** Реализация PK (двумя способами), создание и использование `SEQUENCE` для всех ID.

### ЛР №4: Целостность и оптимизация БД
- **Задачи:** Настройка `FOREIGN KEY` (CASCADE), создание индексов на текстовые поля, применение `CHECK` ограничений.
- **Контрольное задание:** Добавление поля `SecondName` в `CUSTOMER`.

### ЛР №5: Процедурное программирование (Основы PL/pgSQL)
- **Задачи:** Создание хранимой процедуры `CustomerInsert`.
- **Логика:** Проверка существования клиента, использование курсоров (`CURSOR`) для связки клиента с художниками.

### ЛР №6: Управление транзакциями в процедурах
- **Задачи:** Создание процедуры `NewCustomerWithTransaction`.
- **Логика:** Обработка покупки картины, проверка наличия (`IS NULL`), жесткое условие `ROLLBACK` при ошибках.

### ЛР №7: Триггеры автоматизации данных
- **Задачи:** Реализация `BEFORE` триггера (расчет `AskingPrice`) и `AFTER` триггера (авто-регистрация в `TRANSACTION`).

### ЛР №8: Замещающие триггеры и Исключения
- **Задачи:** Создание `INSTEAD OF` триггера для обновления `VIEW`. 
- **Безопасность:** Использование блоков `EXCEPTION` для обработки системных ошибок.
