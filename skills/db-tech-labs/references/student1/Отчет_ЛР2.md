### 1. Создание таблиц и их наполнение (DDL и DML)
В соответствии с заданием была разработана структура учебной базы данных, включающая информацию о студентах, читаемых курсах и результатах экзаменов. Были определены ограничения целостности (PRIMARY KEY, REFERENCES).

**Код создания структур:**
```sql
CREATE TABLE courses (
    c_no VARCHAR(10) PRIMARY KEY, 
    title VARCHAR(100), 
    hours INT
);

CREATE TABLE students (
    s_id INT PRIMARY KEY, 
    name VARCHAR(50), 
    start_year INT
);

CREATE TABLE exams (
    s_id INT REFERENCES students(s_id), 
    c_no VARCHAR(10) REFERENCES courses(c_no), 
    score INT, 
    PRIMARY KEY(s_id, c_no)
);
```

Далее таблицы были наполнены тестовыми данными (по 3-4 записи в каждую) с использованием команды `INSERT INTO ... VALUES ...`.

### 2. Выборка данных и соединения (JOIN)
Были выполнены различные варианты запроса `SELECT` для извлечения данных:
- **Простая выборка:** `SELECT * FROM courses;`
- **Выборка уникальных значений:** `SELECT DISTINCT start_year FROM students;`
- **Внутреннее соединение (INNER JOIN):** 
  Извлечение имен студентов и их оценок по конкретному курсу (C1):
  ```sql
  SELECT s.name, e.score 
  FROM students s 
  JOIN exams e ON s.s_id = e.s_id 
  WHERE e.c_no = 'C1';
  ```
- **Внешнее соединение с группировкой (LEFT JOIN + GROUP BY):**
  Расчет среднего балла по каждой дисциплине (с включением дисциплин без оценок):
  ```sql
  SELECT c.title, AVG(e.score) 
  FROM courses c 
  LEFT JOIN exams e ON c.c_no = e.c_no 
  GROUP BY c.title 
  HAVING AVG(e.score) > 3;
  ```

### 3. Работа с транзакциями (TCL)
Для демонстрации работы транзакций (свойств ACID) была введена дополнительная таблица групп и новое поле в таблице студентов. Операции добавления группы и привязки к ней студентов должны выполняться атомарно.

```sql
CREATE TABLE groups (
    g_no VARCHAR(10) PRIMARY KEY, 
    head_s_id INT NOT NULL
);
ALTER TABLE students ADD COLUMN g_no VARCHAR(10);

-- Начало транзакции
BEGIN;
INSERT INTO groups VALUES ('A-101', 3);
UPDATE students SET g_no = 'A-101';
-- Фиксация изменений
COMMIT;
```
В ходе выполнения лабораторной работы была открыта вторая консоль `psql` для проверки изоляции транзакций. Было установлено, что до выполнения `COMMIT` во второй сессии данные изменения не видны (PostgreSQL использует механизм MVCC).

### 4. Представления (VIEWS)
Для упрощения доступа к часто используемым сложным запросам было создано представление `high_scores`, инкапсулирующее JOIN трех таблиц и фильтрацию отличников:
```sql
CREATE VIEW high_scores AS 
SELECT s.name, c.title, e.score 
FROM students s 
JOIN exams e ON s.s_id=e.s_id 
JOIN courses c ON e.c_no=c.c_no 
WHERE e.score = 5;
```
