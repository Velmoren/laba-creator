# Навык «ОСРВ Ubuntu»

Этот навык используется как дополнение к `laba-creator` для генерации контента по администрированию Ubuntu Linux.

## 1. Спецификации лабораторных работ
Все работы (ЛР 1-7) должны строго соответствовать методическим указаниям из `references/ubuntu_guidelines.md`.

**Темы:**
- **ЛР №1:** Файловая структура (mkdir, ln, vim, nano).
- **ЛР №2:** Архивация (tar, gzip, bzip2).
- **ЛР №3:** Пользователи и права (passwd, groupadd, adduser, chown, chmod, sudo).
- **ЛР №4:** Текстовые файлы (grep, tail, sort, awk).
- **ЛР №5:** Программирование bash (11 уникальных скриптов).
- **ЛР №6:** Процессы (ps, top, renice, kill) и расписание (cron).
- **ЛР №7:** Монтирование (mount, umount, df, du).

## 2. Параметры ОСРВ (Ubuntu)
- **username:** Имя пользователя в системе.
- **machinename:** Название хоста (компьютера).
- **prompt:** Стандартный промпт Ubuntu: `username@machinename:~$ ` (или `# ` для root через sudo).
- **terminal_style:** Тёмно-фиолетовый или чёрный фон, системные шрифты (Ubuntu Mono, Monospace).

## 3. Стандарты оформления отчетов (ОСРВ)
- **Визуализация (КРИТИЧНО):** Все ключевые действия (ввод команд, редактирование файлов, вывод системных утилит) должны сопровождаться ссылками на скриншоты в формате `![Описание](screenshot_N.png)`.
- **Промпты:** Для каждой ссылки `screenshot_N.png` должен быть создан файл `prompt_screenshot_N.txt` объемом не менее 500 символов с детальным описанием интерфейса Ubuntu и логом терминала.
- **Синхронизация данных (КРИТИЧНО):** Содержимое промпта (вводимые команды, пути к файлам, имена пользователей, результаты вывода) ОБЯЗАНО строго соответствовать тексту отчета. Любые противоречия между описанием в отчете и логом в промпте недопустимы.
- **Текстовое наполнение:** Минимум 2 страницы текста (введение, ход работы, технический анализ, заключение).
- **Уникальность контента (КРИТИЧНО):** Запрещено использование одинаковых текстовых блоков (введений, анализов, выводов) для разных студентов. Весь текст отчета должен генерироваться индивидуально, опираясь на `student_settings.json`:
    *   **Стиль изложения (`writing_style`):** Если у студента "Academic Formal", текст должен быть сухим и научным; если "Detailed Technical Diary" — повествование должно идти от первого лица с описанием эмоций и мелких трудностей.
    *   **Детализация (`verbosity_level`):** Для "Advanced" — глубокое погружение в системные вызовы и архитектуру ядра, для "Beginner" — упор на базовые пояснения.
    *   **Технические привычки (`cli_habits`):** Студент, использующий сложные пайпы, должен описывать их в анализе как предпочтительный метод работы.
    *   **Вариативность теории:** Теоретические введения должны фокусироваться на разных аспектах ОС (один студент пишет про VFS, другой про иноды, третий про стандарты POSIX), чтобы исключить текстовые совпадения.
- **Наличие исходных файлов (Source):** Все файлы, упоминаемые в отчете как созданные или измененные (bash-скрипты, программы на Си, конфигурационные файлы, текстовые данные), ОБЯЗАНЫ физически присутствовать в подпапке `source/` соответствующей лабораторной работы. Содержимое этих файлов должно в точности соответствовать листингам из отчета.

*Все общие правила оформления, работы с `source/` и скриншотами наследуются из главного скилла `laba-creator`.*
