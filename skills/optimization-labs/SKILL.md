# Навык «Методы оптимизации»

Этот навык используется для автоматизации решения задач линейного и нелинейного программирования на базе MS Excel (надстройка "Поиск решения" / Solver) для бакалавров по направлению 27.03.04 «Управление в технических системах».

## 1. Теоретическая база и этапы выполнения

**Этапы выполнения лабораторной работы:**
1. **Постановка задачи:** Формулируется первоначальная задача, изучаются факторы, влияющие на процесс.
2. **Построение математической модели:** 
   В самом общем случае математическая модель имеет вид: найти $\max E = \max f(x, y)$ при $g_i(x, y) \le b_i, i=\overline{1,m}$, где:
   - $E = f(x, y)$ – целевая функция (показатель качества или эффективности);
   - $\bar{x}$ – вектор управляемых переменных;
   - $\bar{y}$ – вектор неуправляемых переменных;
   - $g_i$ – функция потребления $i$-го ресурса;
   - $b_i$ – величина (запас) $i$-го ресурса.
3. **Нахождение метода решения:** Применяются методы теории оптимальных решений (Линейное, Нелинейное, Динамическое, Геометрическое, Стохастическое, Дискретное или Эвристическое программирование).
4. **Проверка и корректировка модели:** Оценка адекватности модели реальному процессу путем сравнения предсказанного поведения с фактическим.

## 2. Структура отчета и требования

В каждой конкретной ситуации агент обязан использовать параметры из **`student_settings.json`** для обеспечения уникальности:
- **`excel_language`**: Если `Russian`, использовать в тексте и промптах "=СУММПРОИЗВ()", если `English` - "=SUMPRODUCT()".
- **`math_notation`**: Если `Index`, писать $x_1, x_2$, если `Alphabet` - $x, y$, если `Semantic` - использовать осмысленные имена (например, $P_{max}$).
- **`excel_cell_style`**: Отражать в промптах: `Corporate` (синий/белый), `Colorful` (ярко-зеленый/желтый) или `Minimalist` (без заливки).
- **`excel_layout`**: Располагать блоки на листе вертикально или горизонтально согласно настройке.

**Полный отчет по лабораторной работе должен содержать:**
1. **Титульный лист** (с указанием всех необходимых реквизитов).
2. **Постановку задачи** (точный перенос текста задания).
3. **Математическую модель** с необходимыми комментариями по ее элементам (расписать векторы $x, y$, функцию $f$, ограничения $g$ и неотрицательность переменных).
4. **Описание компьютерной информационной технологии** получения оптимального решения (как именно данные вводились в Excel).
5. **Скриншоты из MS Excel** (вставляются через промпты):
   - Фрагмент исходного рабочего листа Excel.
   - Диалоговое окно «Поиск решения».
   - Диалоговое окно «Параметры поиска решения».
   - Диалоговое окно «Результаты поиска решения».
   - Фрагмент рабочего листа Excel, содержащий результаты решения и (или) фрагмент Отчета по результатам.
6. **Предложения (рекомендации)** лицу, ответственному за принятие решений (ЛПР), по оптимальному управленческому поведению.

## 6. Алгоритм материализации
Агент должен сгенерировать не только Markdown-текст отчета (с пошаговыми математическими выкладками), но и физически создать директории `source/` с промптами для скриншотов Excel (`prompt_excel_1.txt` и т.д.), где будет предельно подробно (до ячейки) расписан интерфейс надстройки "Поиск решения" с установленными целевыми ячейками и ограничениями.

## 7. Обязательная валидация файлов (Excel и Промпты)
Агент обязан проводить инструментальную валидацию всех сгенерированных файлов в директориях `source/` после их создания.
Процесс валидации должен включать:
1. **Excel файлы (`.xlsx`):**
   - Проверку того, что файл успешно открывается (не поврежден и является валидным архивом/документом Excel) через `openpyxl`.
   - Проверку наличия реальных данных в ячейках (файл не должен быть "пустышкой" или содержать только текстовую строку).
2. **Файлы промптов (`.txt`):**
   - Проверку размера каждого текстового промпта. Файл-промпт обязан содержать **не менее 500 символов** детализированного текста. В случае обнаружения "пустышек" или коротких отписок (менее 500 символов) — немедленно их пересоздать.
Для валидации рекомендуется использовать специализированные скрипты. В случае обнаружения пустых или поврежденных файлов, агент должен исправить их до подачи отчета.
