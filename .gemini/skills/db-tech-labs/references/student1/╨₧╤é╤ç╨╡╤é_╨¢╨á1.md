# Отчет по лабораторной работе №1
## Дисциплина: «Технологии баз данных»

**Цель работы:** 
Знакомство с процессом установки и базовой настройкой СУБД PostgreSQL от компании PostgresPro. Создание первого экземпляра базы данных, приобретение практических навыков работы в консольном клиенте SQL Shell (psql) и графической оболочке PgAdmin. Освоение базовых принципов создания пользователей и разграничения прав доступа.

---

### 1. Теоретическая часть и установка экземпляра СУБД
В ходе работы был скачан дистрибутив СУБД PostgreSQL версии 15.x для 64-разрядной ОС. Процесс установки производился с параметрами по умолчанию:
- Порт подключения по умолчанию: `5432`.
- Пользователь СУБД по умолчанию: `postgres`. 
- Пароль суперпользователя: `postgres`.
- Служба Windows/Linux зарегистрирована как `postgresql-15`.

После успешной установки было проверено наличие конфигурационных файлов в каталоге `data`:
- `postgresql.conf` — основной конфигурационный файл сервера (сетевые адреса, порты, параметры памяти).
- `pg_hba.conf` — конфигурация клиентской аутентификации (разрешенные хосты, базы, пользователи и методы шифрования паролей).

В Диспетчере задач (или мониторе процессов) был проанализирован процесс `postgres` (или служба `postgresql-15`). Остановка и запуск службы производились с помощью встроенных утилит `Stop Server` и `Start Server`, что позволило наглядно проследить за выделением оперативной памяти до и после запуска сервера БД.

---

### 2. Работа в командной строке SQL Shell (psql)
Для подключения к серверу баз данных был использован терминальный клиент `psql`. Подключение осуществлено под учетной записью суперпользователя `postgres`.

**Ход выполнения команд в консоли:**
1. **Создание базы данных:**
   Для создания новой БД была выполнена команда:
   `CREATE DATABASE test;`
2. **Подключение к БД:**
   Переход к контексту созданной БД произведен с помощью мета-команды `psql`:
   `\c test`
3. **Создание роли (пользователя) и выдача прав:**
   Был создан новый пользователь `test_user` с паролем `qwerty`:
   `CREATE USER test_user WITH PASSWORD 'qwerty';`
   Затем пользователю были выданы все права на созданную базу:
   `GRANT ALL PRIVILEGES ON DATABASE test TO test_user;`
4. **Проверка созданных объектов:**
   - Список баз данных: `\l`
   - Список ролей: `\dg`

### 3. Использование GUI PgAdmin
Для более удобного визуального администрирования был установлен клиент PgAdmin 4.
1. Осуществлено подключение к локальному серверу `PostgreSQL 15`.
2. Через графический интерфейс (дерево объектов слева) была создана база данных `test1`.
3. В разделе Login/Group Roles создан пользователь `test_user1`.
4. Был опробован встроенный инструмент проектирования **ERD Tool** (Entity Relationship Diagram Tool), позволяющий визуально создавать таблицы и связи между ними, а затем генерировать DDL-код для их физического применения в базе данных.
