# Спецификации лабораторных работ по FreeBSD

В этом файле описаны конкретные требования и шаги для выполнения 7 лабораторных работ по дисциплине "Операционные системы реального времени" (трек FreeBSD).

**Важное замечание по интерфейсу FreeBSD:** 
Интерфейс FreeBSD "из коробки" — это чистая текстовая консоль. Черный экран, белый шрифт. Промпт пользователя — это просто `$ `. Промпт суперпользователя (root) — это просто `# `. Цветовая подсветка команд или файлов по умолчанию отключена. При генерации промптов для скриншотов делайте акцент именно на этой минималистичности. Картинка должна состоять только из текста команд, кода и ответов системы. Никаких красивых окошек, кнопок закрытия окна или полупрозрачных фонов быть не должно, если только студент явно не использует удаленный SSH-терминал с кастомным фоном.

## ЛР №1: Файловая структура и команды работы с файлами
- **Суть:** Создание текстовых файлов в редакторах `vim` и `nano`. Построение заданного дерева каталогов с помощью `mkdir`. Создание жестких и символических ссылок с помощью `ln` и `ln -s`. Демонстрация работы ссылок после удаления оригинального файла (символическая ломается, жесткая продолжает работать).
- **Требуемые скриншоты (минимум 4):** 
  1. Работа в текстовом редакторе: Процесс ввода текста в `vim` или `nano`.
  2. Создание структуры: Использование `mkdir -p` для создания дерева каталогов и `ls -R` для проверки.
  3. Создание ссылок: Команды `ln` (жесткая) и `ln -s` (символическая), вывод `ls -li` для проверки inode.
  4. Удаление и проверка: Команда `rm` оригинала и последующая попытка прочитать содержимое через ссылки (`cat link_name`), демонстрирующая работоспособность жесткой и ошибку символической.

## ЛР №2: Архивация и сжатие файлов
- **Суть:** Использование утилиты `tar` для создания архивов. Нужно создать 3 типа архивов: из разных каталогов, из одного каталога, со всеми подкаталогами. Проверка размера с помощью `ls -l`. Сжатие одного из tar-архивов с помощью `gzip` и `bzip2`, сравнение размеров получившихся файлов. Распаковка архива.
- **Требуемые скриншоты (минимум 4):** 
  1. Создание архива: Процесс выполнения `tar -cvf archive.tar source_dir`.
  2. Использование масок: Добавление файлов по маске, например `tar -rvf archive.tar *.txt`.
  3. Сжатие файлов: Выполнение команд `gzip` и `bzip2` над архивами.
  4. Сравнение размеров: Вывод `ls -lh` для сравнения `tar`, `gz` и `bz2` версий.

## ЛР №3: Управление пользователями, группами и правами
- **Суть:** Изучение системных файлов `/etc/passwd` и `/etc/group`. Использование `su` для перехода в режим root. Создание нового пользователя (`adduser`), проверка создания его домашнего каталога (`/home/...`). Смена пароля (`passwd`) и полного имени (`chpass`). Создание новой группы (`groupadd` или `pw groupadd`), добавление в нее пользователя. Использование `id` для проверки. Изменение владельца файла (`chown`) и прав доступа: сначала в буквенном виде (`chmod a+rx`), затем в числовом (`chmod 755`). Удаление пользователя (`rmuser` или `userdel`).
- **Требуемые скриншоты (минимум 4):**
  1. Исследование системы: Просмотр `/etc/passwd` и `/etc/group` через `tail` или `grep`.
  2. Создание пользователя: Использование `adduser` (диалог создания) и переход в `root` через `su -`.
  3. Управление группами: Создание группы (`pw groupadd`) и добавление пользователя (`pw groupmod ... -m ...`), проверка через `id`.
  4. Права доступа: Демонстрация `chmod` (буквенный и числовой режимы) и `chown` с проверкой через `ls -l`.

## ЛР №4: Текстовые файлы и утилита grep
- **Суть:** Выполнение серии фильтраций с использованием `grep` по тестовым файлам (query1-query4). Выборки по коду, индексу, цене, имени и фамилии продавца. Использование команд `tail` (вывод с конца, вывод символов `tail -c`), вывод строк в обратном порядке (через пайпы или утилиту `tac`/`tail -r`), сортировка `sort` последних строк. Подсчет количества подключенных пользователей в системе (например, `who | wc -l`).
- **Требуемые скриншоты (минимум 4):** 
  1. Фильтрация `grep`: Выполнение сложных запросов с использованием регулярных выражений.
  2. Работа с концом файла: Команды `tail -n` и `tail -c` на больших текстовых файлах.
  3. Комбинирование команд: Конвейер (pipe) `grep | sort | uniq` или аналогичные.
  4. Статистика системы: Команды `who`, `last` в сочетании с `wc -l` для подсчета данных.

## ЛР №5: Программный интерпретатор bash
- **Суть:** Написание 11 коротких bash-скриптов:
  1. Вывод параметров командной строки (`$1`, `$2`...).
  2. Арифметические вычисления (`D=(A*2 + B/3)*C`).
  3. Сохранение списка файлов домашнего каталога в файл, подсчет количества (`ls | wc -l`).
  4. Запрос имени пользователя (`read`) и проверка на совпадение с `$USER`.
  5. Запрос имени файла и вывод его типа через утилиту `file`.
  6. Вывод файлов, созданных в один день (`ls -l | grep ...` или `find`).
  7. Проверка, является ли файл ссылкой (`-h` или `file`), и прохождение по цепочке ссылок (через `readlink`).
  8. Подсчет количества строк с заданным словом (`grep -c`) с предварительной проверкой типа файла (только для "text").
  9. Поиск жестких ссылок на файл по его inode (`ls -i` или `find -samefile`).
  10. Подсчет файлов пользователя (`find -user`) и всех остальных.
  11. Парсинг переменной окружения `$PATH` и вывод прав доступа для каждой директории оттуда (`ls -ld`).
- **Требуемые скриншоты (минимум 4):** Групповое выполнение скриптов:
  1. Скрипты 1-3: Параметры, арифметика и работа с файлами.
  2. Скрипты 4-6: Интерактивный ввод, типы файлов (`file`) и фильтрация по дате.
  3. Скрипты 7-9: Ссылки, inodes и проверка типов файлов в циклах.
  4. Скрипты 10-11: Поиск файлов по владельцу и парсинг `$PATH`.

## ЛР №6: Мониторинг производительности и процессы
- **Суть:** Изучение утилит `top` и `ps` для просмотра процессов. Использование команды `renice` для понижения приоритета (увеличения номера nice) своего процесса. Создание программы на Си (`infinite_loop.c`), ее компиляция (`gcc`) и запуск в фоновом режиме с амперсандом `&`. Остановка процесса через `kill`. Написание bash-скрипта архивации (если папка не пуста, жмем `tar` с датой в имени, кладем в другую папку) и добавление его в `cron` (`crontab -e`) для запуска каждые 2 минуты. Удаление сценария из `cron`.
- **Требуемые скриншоты (минимум 5):**
  1. Мониторинг: Активные процессы в `top` и дерево процессов в `ps -axjf`.
  2. Приоритеты: Изменение приоритета процесса через `renice`.
  3. Управление фоновыми процессами: Компиляция `gcc`, запуск с `&`, просмотр через `jobs`.
  4. Завершение работы: Использование `kill` для остановки фонового процесса.
  5. Планировщик: Редактирование `crontab -e` и демонстрация логов или созданных архивов в результате работы `cron`.

## ЛР №7: Монтирование устройств
- **Суть:** Практика монтирования файловых систем. Монтирование USB-накопителя (`mount`), копирование файлов, размонтирование (`umount`). Монтирование CD-диска (режим только для чтения). Проверка свободного места командой `df` и размера директорий командой `du`.
- **Требуемые скриншоты (минимум 4):**
  1. Монтирование: Команда `mount` для USB и проверка через `ls`.
  2. Работа с данными: Копирование файлов на внешнее устройство и `sync`.
  3. Информация о дисках: Вывод `df -h` для всей системы.
  4. Анализ места: Использование `du -sh` для конкретных директорий и `umount` для завершения работы.
