# Генератор лабораторных работ «Операционные системы реального времени» (ОСРВ)

Этот навык предназначен для генерации уникальных пакетов лабораторных работ (Отчеты + bash-скрипты/программы + промпты для скриншотов) по дисциплине «Операционные системы реального времени» (Трек администрирования FreeBSD, 7 лабораторных работ).

## 1. Входные параметры и метаданные

При генерации необходимо определить две ключевые переменные: **username** (имя пользователя) и **machinename** (название компьютера). Они используются в текстах отчетов, путях к файлам и в промптах для скриншотов (например, `username@machinename:~$`).

**Приоритет определения переменных:**
1.  **Явное указание:** Если переменные переданы в текущем запросе.
2.  **Парсинг папки:** Если не указаны, извлечь из названия корневой директории студента в `src/`. Формат названия: `[ID]_[username]_[machinename]` (например, из `2_trushkin_macbus` получаем `trushkin` и `macbus`).
3.  **Рандом:** Если папка называется просто `[ID]`, сгенерировать реалистичные случайные имена (избегайте имен вроде `student1`, используйте `dev`, `admin`, `workstation`, `mac-pro` и т.д.).

Дополнительные переменные:
1.  **Операционная система студента:** (Windows, macOS или Linux). Влияет на внешний вид окна терминала в промптах скриншотов.
2.  **Фон рабочего стола:** (оттенок цвета или рисунок). Добавляется в промпты скриншотов. **Ограничение: никаких элементов рабочего стола (иконок, панелей задач) кроме самого окна терминала быть не должно.**

## 2. Спецификации лабораторных работ

Все 7 лабораторных работ должны строго соответствовать методическим указаниям.
Ознакомьтесь с файлом `references/freebsd_guidelines.md` для получения подробного списка требований к каждой ЛР, включая список обязательных скриншотов.

Краткий перечень тем:
- **ЛР №1:** Файловая структура (mkdir, ln, vim).
- **ЛР №2:** Архивация (tar, gzip, bzip2).
- **ЛР №3:** Пользователи и права (passwd, group, adduser, chown, chmod).
- **ЛР №4:** Текстовые файлы (grep, tail, sort).
- **ЛР №5:** Программирование bash (11 скриптов).
- **ЛР №6:** Процессы (ps, top, renice, kill) и расписание (cron).
- **ЛР №7:** Монтирование (mount, umount, df, du).

## 3. Алгоритм работы (Генерация пакета)

Для генерации пакета создайте структуру по пути: `src/[Полное_имя_папки_студента]/Операционные системы реального времени/ЛР_[Номер]/`.

1. **Файл отчета (`Отчет_ЛРX.md`):** 
   - Структура: Введение/Цель, Ход работы (с командами и описаниями), Выводы. 
   - **Максимизация контента:** Описывайте ход работы максимально подробно. Добавляйте теоретические пояснения к командам, развернутые описания каждого шага и объемные выводы. Большой объем текста помогает сделать уникальность (различия) вариантов более очевидной.
   - **Персонализация:** Используйте определенные выше *username* и *machinename* в путях (например, `/usr/home/username/`) и в описании команд.
   - *Титульные листы и контрольные вопросы не генерировать.*
2. **Файлы скриптов (`.sh` или `.c`):** 
   - Особенно актуально для ЛР5 (11 bash-скриптов) и ЛР6 (скрипт архивации по cron и программа на Си с бесконечным циклом). 
   - Код скриптов должен быть рабочим и соответствовать стилю студента.
3. **Промпты для скриншотов (`prompt_screenshot_X.txt`):**
   - Сгенерируйте промпты для ИИ (на английском языке) на основе требований из `freebsd_guidelines.md`.
   - **Детализация:** Промпты должны быть максимально подробными. Описывайте каждую визуальную деталь: шрифт (Consolas/Menlo), цвет текста, точное расположение элементов, наличие или отсутствие теней и прозрачности окна, точный текст (весь код и вывод). Чем детальнее промпт, тем точнее сработает генератор изображений.
   - Внедрите переменные **OS** и **Фон рабочего стола**.
   - **Реализм:** В приглашении командной строки FreeBSD обязательно используйте `username@machinename` (если не указано иное в задании). Избегайте шаблонных имен типа `student_1`.
   - **КРИТИЧЕСКОЕ ТРЕБОВАНИЕ К ВНЕШНЕМУ ВИДУ:** По умолчанию интерфейс FreeBSD представляет собой "голую" консоль: черный фон, белый моноширинный шрифт (VGA 80x25). Стандартный промпт обычного пользователя: `$ ` (просто знак доллара и пробел). Промпт суперпользователя: `# `. Никаких цветов, путей или хостнеймов по умолчанию нет.
   - Добавьте строгое ограничение в каждый промпт: `The image MUST look like a raw, minimalist FreeBSD terminal console. Solid black background, crisp white monospaced font. The terminal prompt should just be "$ " for a normal user or "# " for root. Do NOT generate any desktop environment elements, window borders, icons, taskbars, or colorful text. Focus 100% on the text, commands, and code structure.`
   - **Фон:** Окно терминала должно располагаться на слегка размытом фоне из переменной "Фон рабочего стола".
   - В промпте должен быть явно указан текст, который должен быть виден в терминале (команды и реалистичный вывод ОС FreeBSD). Имя пользователя в терминале должно соответствовать параметру *username* (если по ходу ЛР настраивается кастомный промпт).
