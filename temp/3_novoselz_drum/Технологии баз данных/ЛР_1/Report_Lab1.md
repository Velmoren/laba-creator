# Лабораторная работа №1. Отчет
**Дисциплина:** Технологии баз данных
**Тема:** Развертывание экземпляра PostgreSQL, конфигурация и управление правами доступа.

## 1. Введение и постановка задачи
Основной задачей данной лабораторной работы являлось изучение архитектуры PostgreSQL, получение практического опыта инсталляции сервера БД, а также освоение синтаксиса встроенных утилит командной строки для первоначальной инициализации рабочего пространства. В ходе выполнения предполагалось создание новой базы данных, настройка нового пользователя и выдача ему соответствующих привилегий, исключающих использование учетной записи суперпользователя для рутинных операций.

## 2. Ход выполнения работы

### 2.1. Установка программного обеспечения
Был загружен и установлен дистрибутив `PostgreSQL 15.x`. В мастере установки были оставлены порты по умолчанию (`5432`). В качестве директории хранения данных (Data Directory) был указан путь `C:\Program Files\PostgreSQL\15\data`.
Также была проведена инспекция системных служб. Служба `postgresql-15` работает в фоновом режиме, обеспечивая бесперебойный доступ к серверу. Через оснастку "Службы" ОС Windows был произведен тестовый перезапуск сервиса, что подтвердило штатную работу демона.

### 2.2. Инициализация базы данных и ролей
В качестве основного инструмента взаимодействия применялась утилита `psql`. Подключение было выполнено с использованием реквизитов учетной записи `postgres`. 
Был написан и выполнен следующий SQL-скрипт (DDL/DCL команды):

```sql
/* Создание рабочей базы данных */
CREATE DATABASE practice_db;

/* Подключение к созданной базе */
\c practice_db

/* Создание администратора базы данных */
CREATE USER db_admin WITH ENCRYPTED PASSWORD 'admin123';

/* Передача прав владения БД новому пользователю */
ALTER DATABASE practice_db OWNER TO db_admin;
```

Для проверки успешности выполнения операций были задействованы встроенные slash-команды `psql`:
- Вызов `\l` отобразил `practice_db` в списке доступных каталогов, где владельцем значится `db_admin`.
- Вызов `\du` вывел список системных ролей, подтвердив успешное создание `db_admin` с правом логина.

### 2.3. Графический клиент PgAdmin
Параллельно был настроен доступ через графическую среду `PgAdmin 4`. В интерфейсе было установлено подключение к `localhost:5432`. База данных `practice_db` успешно отобразилась в панели Object Browser. Были изучены вкладки Properties, SQL и Statistics, показывающие метаданные объектов в реальном времени.
