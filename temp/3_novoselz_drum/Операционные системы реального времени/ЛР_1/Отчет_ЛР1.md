## 1. Теоретическая база

Основой FreeBSD является файловая иерархия, которая значительно отличается от дисковой модели Windows. Здесь нет дисков C: или D:, а все устройства монтируются в единое дерево.

### 1.1. Стандарты иерархии
- `/boot` — критические файлы для старта системы (ядро `kernel`).
- `/root` — домашняя папка администратора.
- `/usr/home` — место для пользовательских данных.
- `/tmp` — временное хранилище, часто очищаемое при перезагрузке.

### 1.2. Текстовое редактирование
В FreeBSD поставляется классический редактор `vi`. Он требует времени на освоение, так как разделяет ввод текста и ввод команд. Современной альтернативой является `ee` (Easy Editor), который более интуитивен для пользователей Windows.

### 1.3. Концепция ссылок
- **Hard Link:** Дополнительное имя для одного и того же физического участка данных. Не занимает лишнего места.
- **Soft Link (Symlink):** Ярлык, содержащий путь к файлу. Удобен для создания ссылок на динамически меняющиеся пути.

## 2. Процесс выполнения

### 2.1. Создание каталогов
Я подготовил рабочую среду в своей домашней директории.

```bash
novoselz@drum $ mkdir -p ~/osrv_labs/lab1/archive ~/osrv_labs/lab1/sources
novoselz@drum $ cd ~/osrv_labs/lab1
```

### 2.2. Редактирование файлов
С помощью редактора `vim` я создал конфигурационный файл `config.sys` в папке `sources`.

```bash
novoselz@drum $ vim sources/config.sys
```
*Я ввел несколько строк параметров, сохранил изменения через :wq.*

### 2.3. Манипуляции со ссылками
Создадим исходный файл и проверим работу ссылок.

```bash
novoselz@drum $ echo "Data v1.0" > sources/main.dat
novoselz@drum $ ln sources/main.dat archive/backup_hard
novoselz@drum $ ln -s /home/novoselz/osrv_labs/lab1/sources/main.dat archive/backup_soft
```

Просмотр результатов:
```bash
novoselz@drum $ ls -li sources/ archive/
```

### 2.4. Удаление и проверка
Я удалил основной файл, чтобы убедиться в устойчивости жесткой ссылки.

```bash
novoselz@drum $ rm sources/main.dat
novoselz@drum $ cat archive/backup_hard
# Данные сохранились.
novoselz@drum $ cat archive/backup_soft
# Ошибка: файл не найден (битая ссылка).
```

## 3. Заключение

В рамках первой лабораторной работы я освоил базовую навигацию в FreeBSD. Главным открытием стала система ссылок, которая позволяет гибко управлять доступом к данным. В отличие от ярлыков Windows, жесткие ссылки в BSD обеспечивают надежность хранения данных даже при удалении основного имени файла.
