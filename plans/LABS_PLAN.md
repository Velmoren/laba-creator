# ГЕНЕРАЛЬНЫЙ ПЛАН: ГЕНЕРАЦИЯ ПАКЕТОВ ЛАБОРАТОРНЫХ РАБОТ

## 1. Концепция Конвейера
- **Источник**: `src/[ID]/лабораторные/` (папки с MD и медиа-данными).
- **Движок**: `scripts/generate_labs.js` (автоматическая сборка Word + копирование доп. материалов).
- **Результат**: `dist/[ID]/labs/` (готовые папки для выдачи студентам).

## 2. Спецификация пакета ЛР
Каждая работа собирается как отдельная папка, включающая:
1.  **Инструкция (Word)**: Генерируется из `Методичка.md`.
2.  **Вспомогательные файлы**: Скрипты, тестовые данные, исходные коды из соответствующих подпапок `src`.
3.  **Контроль версий**: Каждый заход (ID) имеет свою изолированную среду.

## 3. Логика Сборки
1.  Вызов: `node scripts/generate_labs.js [ID]`.
2.  Скрипт проходит по всем подпапкам в `src/[ID]/лабораторные/`.
3.  Для каждой подпапки:
    - Генерирует `Методичка_ЛР_X.docx`.
    - Копирует все остальные файлы из этой подпапки в `dist/[ID]/labs/LR_X/`.

## 4. Контроль качества
- [ ] Однотипность оформления во всех 7 работах.
- [ ] Наличие таблиц с вариантами (10 шт.).
- [ ] Наличие контрольных вопросов в конце документа.
- [ ] Работоспособность приложенных скриптов и данных.
