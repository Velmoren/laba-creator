## Текстовые требования

### Шрифт:
- Тип: Times New Roman или аналог
- Размер основного текста: 14 пт
- Размер сносок: 12 пт

### Интервалы:
- Межстрочный интервал: 1.5
- Отступ первой строки абзаца: 1.25 см

### Выравнивание:
- Основной текст: по ширине
- Заголовки: по центру или по левому краю

---

## Нумерация страниц

- Страницы нумеруются арабскими цифрами
- Номер страницы проставляется внизу страницы по центру
- Титульный лист включается в общую нумерацию, но номер не ставится

## Заголовки

### Структура заголовков:

```
ГЛАВА 1. НАЗВАНИЕ ГЛАВЫ          ← Заголовок главы (прописные, полужирный)
                                 ← Пропуск строки

1.1. Название параграфа          ← Заголовок параграфа (прописные/строчные, полужирный)
                                 ← Пропуск строки

Текст параграфа...
```

### Правила:
- Точка в конце заголовка НЕ ставится
- Перенос слов в заголовках НЕ допускается
- Заголовки отделяются от текста пропуском строки

---

## Иллюстрации (Рисунки)

### Требования:
- Все рисунки нумеруются
- Подпись под рисунком: `Рисунок 1 – Название`
- Рисунки размещаются по тексту или в приложениях
- Должны быть ссылки на рисунки в тексте

### Пример:
```
...структура базы данных представлена на рисунке 2.

                          ┌─────────┐
                          │ Клиент  │
                          └────┬────┘
                               │
                               ▼
                          ┌─────────┐
                          │ Заказ   │
                          └─────────┘

        Рисунок 2 – Схема взаимосвязи таблиц
```

## Таблицы

### Техническое правило (Markdown):
> **КРИТИЧЕСКИ ВАЖНО:** Перед началом любой таблицы в Markdown **ОБЯЗАТЕЛЬНО** должна быть пустая строка. Если таблица идёт сразу после текста или заголовка без пустой строки, она не будет распознана конвертером и отобразится как обычный текст.

**Правильно:**
```markdown
Текст перед таблицей.

| Столбец 1 | Столбец 2 |
|-----------|-----------|
| Данные 1  | Данные 2  |
```

**Неправильно:**
```markdown
Текст перед таблицей.
| Столбец 1 | Столбец 2 |
|-----------|-----------|
```

### Требования:
- Все таблицы нумеруются
- Над таблицей: `Таблица 1 – Название`
- Заголовки колонок выделяются полужирным
- Должны быть ссылки на таблицы в тексте

### Пример:
```
...данные приведены в таблице 3.

        Таблица 3 – Характеристика сущностей

┌─────────────┬────────────────┬─────────────┐
│  Сущность   │    Описание     │   Ключ      │
├─────────────┼────────────────┼─────────────┤
│   Клиент    │ Покупатель     │ id_клиента  │
│   Заказ     │ Заказ клиента  │ id_заказа   │
└─────────────┴────────────────┴─────────────┘
```

## Формулы

### Требования:
- Нумеруются справа в круглых скобках
- Пояснения символов приводятся под формулой

### Пример:
```
Формула для расчёта стоимости:

        С = К × Ц                              (1)

где С – стоимость заказа;
    К – количество товаров;
    Ц – цена единицы товара.
```

## Список источников

### Требования к оформлению:

**Книга одного автора:**
```
Фамилия И.О. Название книги. – Город: Издательство, Год. – С.
```

**Статья из журнала:**
```
Фамилия И.О. Название статьи // Название журнала. – Год. – №. – С.
```

**Электронный ресурс:**
```
Фамилия И.О. Название [Электронный ресурс]. – Режим доступа: URL
```

### Полный пример списка литературы — см. [07-examples.md](./07-examples.md)

## Сокращения

### Требования:
- Первый раз полное название, затем сокращение в скобках
- Пример: «автоматизированная информационная система (АИС)»

### Список сокращений:
- Оформляется в виде алфавитного перечня
- Размещается перед списком источников

---

## Сноски

- Нумеруются арабскими цифрами
- Размещаются в конце страницы
- Отделяются чертой

---

## Приложения

### Требования:
- Нумеруются арабскими цифрами: «Приложение 1», «Приложение 2»
- Имеют содержательный заголовок
- Начинаются с новой страницы
- В тексте — ссылки на приложения

---

## Общие правила

1. **Объём работы:** не менее 30 страниц
2. **Структура:** титульный лист → содержание → введение → главы → заключение → источники → приложения
3. **Однообразие:** оформление должно быть единообразным
4. **Аккуратность:** без помарок и исправлений
5. **Содержательность:** каждая часть работы должна содержать законченную информацию
