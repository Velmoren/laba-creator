## ЛР1: Файловая структура и команды работы с файлами

**Цель:** Ознакомиться с файловой структурой ОС FreeBSD, изучить основные команды работы с файлами.

### Задания:
1. Создать два текстовых файла в разных текстовых редакторах (vim, nano)
2. Создать структуру каталогов
3. Проверить работу ссылок после удаления оригинала

### Ключевые команды:
```bash
mkdir          # создание каталога
mkdir -p path  # создание с вложенными каталогами
cd             # смена каталога
ls             # список файлов
ls -la         # подробный список
pwd            # текущий каталог
rm             # удаление
rm -r          # удаление каталога
cp             # копирование
mv             # перемещение
vim filename   # редактор vim
nano filename  # редактор nano
ln             # создание жёсткой ссылки
ln -s          # создание символической ссылки
```

### Жёсткие vs Символические ссылки:
```
Жёсткая:  ссылка на inode, удаление оригинала не влияет
Символич: ярлык на файл, удаление оригинала делает ссылку битой
```

## ЛР2: Архивирование и сжатие файлов

**Цель:** Изучить основные команды для архивирования и разархивирования файлов.

### Задания:
1. Создать tar-архив для файлов из разных каталогов
2. Создать tar-архив для всех файлов каталога
3. Создать архив с подкаталогами
4. Проверить, что tar не сжимает
5. Посмотреть содержимое архива, распаковать
6. Сжать gzip и bzip2, сравнить степень сжатия

### Ключевые команды:
```bash
tar -cf archive.tar files     # создание архива
tar -tf archive.tar           # просмотр содержимого
tar -xf archive.tar           # распаковка
tar -czf archive.tar.gz files # создание + gzip
tar -cjf archive.tar.bz2 files # создание + bzip2
tar -xzf archive.tar.gz       # распаковка gzip
tar -xjf archive.tar.bz2      # распаковка bzip2
gzip file                     # сжатие gzip
gunzip file.gz                # распаковка gzip
bzip2 file                    # сжатие bzip2
bunzip2 file.bz2              # распаковка bzip2
```

### Пример:
```bash
# Создать архив из файлов разных каталогов
tar -cf archive.tar /home/user/docs/*.txt /home/user/data/*.csv

# Создать архив с подкаталогами
tar -cf backup.tar /home/user

# Сжать и сравнить
gzip -k archive.tar
bzip2 -k archive.tar
ls -lh archive.tar*
```

## ЛР3: Пользователи, группы и права доступа

**Цель:** Изучить команды работы с пользователями, группами и правами доступа.

### Задания:
1. Посмотреть /etc/passwd и /etc/group
2. Создать пользователя (adduser)
3. Изменить пароль (passwd)
4. Изменить полное имя (chpass)
5. Создать группу (pw groupadd)
6. Добавить пользователя в группу (pw groupmod)
7. Проверить принадлежность (id)
8. Изменить владельца и группу файла (chown, chgrp)
9. Изменить атрибуты файла (chmod буквами и цифрами)
10. Удалить пользователя (rmuser)

### Ключевые команды:
```bash
su username              # смена пользователя
adduser                  # создание пользователя (интерактивно)
passwd [username]        # изменение пароля
chpass username          # изменение информации о пользователе
pw groupadd groupname    # создание группы
pw groupmod -n newname oldname  # переименовать группу
pw groupmod -a user group # добавить пользователя в группу
id [username]            # информация о пользователе
chown user file          # изменить владельца
chown user:group file    # изменить владельца и группу
chgrp group file         # изменить группу
chmod mode file          # изменить права
rmuser username          # удалить пользователя
```

### Формат прав доступа:
```
rwx rwx rwx
│   │   │
│   │   └── права для остальных
│   └────── права для группы
└────────── права для владельца

r = 4 (чтение)
w = 2 (запись)
x = 1 (выполнение)
```

### Примеры chmod:
```bash
chmod u+rwx file           # добавить rwx для владельца
chmod go-rwx file          # убрать все для группы и остальных
chmod 755 file             # rwxr-xr-x
chmod 644 file             # rw-r--r--
chmod 600 file             # rw-------
```

## ЛР4: Работа с текстовыми файлами и утилита grep

**Цель:** Изучить основные команды работы с текстовыми файлами и утилиту grep.

### Задания:
1. Выбрать строки по шаблону (grep)
2. Выбрать строки по началу/концу (regex)
3. Вывести символы/строки с конца файла
4. Вывести строки в обратном порядке
5. Сортировка
6. Подсчёт пользователей (who/w)

### Ключевые команды:
```bash
grep "pattern" file              # поиск по шаблону
grep "^pattern" file             # строки начинающиеся с pattern
grep "word$" file               # строки заканчивающиеся на word
grep -v "pattern" file          # строки НЕ содержащие pattern
grep -i "pattern" file          # без учёта регистра
grep -c "pattern" file          # только подсчёт совпадений
grep -n "pattern" file          # с номерами строк
tail -n N file                  # последние N строк
tail -c N file                  # последние N символов
head -n N file                  # первые N строк
tac file                        # строки в обратном порядке
sort file                       # сортировка
sort -u file                    # уникальные строки
wc -l file                      # подсчёт строк
wc -w file                      # подсчёт слов
wc -c file                      # подсчёт символов
who                             # кто в системе
w                               # подробная информация
```

### Примеры grep:
```bash
grep "ivanov" users.txt         # найти ivanov
grep "^[0-9]" data.txt          # строки начинающиеся с цифры
grep "error$" log.txt           # строки заканчивающиеся на error
grep -E "(cat|dog)" file        # extended regex: cat или dog
```

## ЛР5: Программный интерпретатор bash

**Цель:** Изучить работу с программным интерпретатором bash.

### Задания:
1. Вывод параметров командной строки с номерами ($n)
2. Вычисления с переменными
3. Формирование файла со списком файлов
4. Запрос и сравнение имени пользователя
5. Проверка типа файла (file)
6. Поиск файлов по дате создания
7. Проверка символических ссылок
8. Подсчёт строк со словом (wc)
9. Поиск жёстких ссылок (ls -li)
10. Вывод файлов по владельцу
11. Вывод каталогов из PATH (env, ls -ld)

### Ключевые переменные bash:
```bash
$0              # имя скрипта
$1, $2, ...     # параметры командной строки
$@              # все параметры
$#              # количество параметров
$?              # код завершения предыдущей команды
$$              # PID текущего процесса
```

### Пример скрипта:
```bash
#!/bin/bash

# Вывод параметров командной строки
echo "Имя скрипта: $0"
echo "Параметры:"
i=1
for param in $@; do
    echo "  $i: $param"
    i=$((i + 1))
done

# Вычисления
A=10; B=100; C=200
D=$(( (A*2 + B/3) * C ))
echo "D = $D"

# Проверка текущего пользователя
read -p "Введите имя пользователя: " input_user
if [ "$input_user" = "$USER" ]; then
    echo "Верно!"
else
    echo "Неверно!"
fi

# Список файлов
ls > files_list.txt
wc -l files_list.txt

# Проверка типа файла
file myfile.txt
```

## ЛР6: Мониторинг процессов и автоматизация заданий

**Цель:** Изучение средств для мониторинга производительности, управления процессами и автоматизации заданий.

### Задания:
1. Работа с top и ps
2. Изменение приоритета процесса (renice)
3. Создание и запуск программы на Си в фоновом режиме
4. Прерывание процесса (kill)
5. Создание shell-сценария для архивации
6. Добавление права на выполнение (chmod +x)
7. Создание cron-сценария (каждые 2 минуты)
8. Проверка и удаление cron-сценария

### Ключевые команды:
```bash
top                  # мониторинг процессов в реальном времени
ps                   # список процессов
ps aux               # подробный список всех процессов
ps -ef               # подробный список
renice +N PID        # изменить приоритет (положительное = ниже)
renice -N PID        # отрицательное = выше
command &            # запуск в фоне
jobs                 # список фоновых задач
fg %N                # перенести в фон
bg %N                # перенести на передний план
kill PID             # завершить процесс (SIGTERM)
kill -9 PID          # принудительно завершить (SIGKILL)
chmod +x script.sh   # добавить право на выполнение
crontab -e           # редактирование crontab
crontab -l           # просмотр crontab
crontab -r           # удаление crontab
```

### Формат crontab:
```
┌───────────── минута (0-59)
│ ┌───────────── час (0-23)
│ │ ┌───────────── день месяца (1-31)
│ │ │ ┌───────────── месяц (1-12)
│ │ │ │ ┌───────────── день недели (0-7, 0=воскресенье)
│ │ │ │ │
* * * * * command

Пример: */2 * * * * /home/user/backup.sh
        # каждые 2 минуты
```

### Пример архивационного скрипта:
```bash
#!/bin/bash
# Проверка непустого каталога и архивация

if [ -d "$1" ] && [ "$(ls -A $1)" ]; then
    DATE=$(date +"%Y%m%d_%H%M%S")
    tar -czf "$2/backup_${DATE}.tar.gz" "$1"
    echo "Архив создан: backup_${DATE}.tar.gz"
else
    echo "Каталог пуст или не существует"
fi
```

## ЛР7: Монтирование устройств и проверка состояния

**Цель:** Изучение средств для монтирования различных устройств и проверки состояния.

### Задания:
1. Монтирование флэшки
2. Просмотр содержимого
3. Копирование файлов
4. Монтирование CD-диска
5. Проверка результатов (df)
6. Демонтирование
7. Изучение команды du

### Ключевые команды:
```bash
mount /dev/device /mnt/path  # монтирование
umount /mnt/path             # размонтирование
umount /dev/device           # размонтирование по устройству
df                           # свободное место на дисках
df -h                        # в человекочитаемом формате
du                           # размер каталогов
du -sh directory             # общий размер каталога
du -h --max-depth=1          # размер с глубиной 1
lsblk                        # блочные устройства
dmesg | tail                 # последние сообщения ядра
```

### Пример:
```bash
# Найти устройство
dmesg | grep -i usb
lsblk

# Монтировать флешку
mkdir /mnt/usb
mount /dev/da0s1 /mnt/usb

# Работа с файлами
ls /mnt/usb
cp file.txt ~/documents/

# Проверить место
df -h
du -sh /mnt/usb

# Размонтировать
umount /mnt/usb
```
