## ГОСТ требования

> Требования к оформлению курсовой работы согласно ГОСТ

### Нормативные документы (ГОСТ)

- **ГОСТ 2.105-95** — Единая система конструкторской документации. Общие требования к текстовым документам
- **ГОСТ 2.106-95** — Единая система конструкторской документации. Текстовые документы
- **ГОСТ 2.301-68** — Единая система конструкторской документации. Форматы
- **ГОСТ 7.32-2001** — Система стандартов по информации, библиотечному и издательскому делу. Отчёт о НИР. Структура и правила оформления

### Структура работы (ГОСТ)

| Элемент | Требование |
|---------|------------|
| Титульный лист | Наименование вуза, кафедры, тема, данные студента и руководителя, место и год |
| Содержание (оглавление) | Перечень элементов с указанием номеров страниц |
| Введение | Актуальность, цель, задачи, методы, структура работы |
| Основная часть | 3 главы с выводами по каждой |
| Заключение | Основные результаты, выводы, предложения, рекомендации |
| Список источников | ≥15 источников, алфавитно-хронологическая последовательность |
| Приложения | Нумерация арабскими цифрами, ссылки в тексте |

### Требования к оформлению текста (ГОСТ)

| Элемент | Требование |
|---------|------------|
| Объём | ≥30 страниц машинописного текста |
| Шрифт | Обычный ( Times New Roman 14pt или аналог) |
| Интервал | 1,5 интервала |
| Поля | Левое — 30 мм, правое — 15 мм, верхнее и нижнее — 20 мм |
| Нумерация страниц | Начинается с титульного листа, номер на страницах с 3 |
| Заголовки | Главы — полужирным, нумерация арабскими цифрами |
| Подзаголовки | Нумерация в пределах главы |

### Требования к таблицам (ГОСТ)

| Элемент | Требование |
|---------|------------|
| Название | Над таблицей, слева, после номера |
| Нумерация | Сквозная или в пределах главы (Таблица 1.1) |
| Заголовки | Заголовок граф — единственное число |
| Разделители | Горизонтальные линии в заголовке и конце таблицы |
| Ссылки | Обязательная ссылка в тексте перед таблицей |
| Повтор заголовка | При переносе на другую страницу |

### Требования к рисункам (ГОСТ)

| Элемент | Требование |
|---------|------------|
| Название | Под рисунком, по центру, после номера |
| Нумерация | Сквозная или в пределах главы (Рисунок 1.1) |
| Качество | Чёткое изображение, штриховка различима |
| Ссылки | Обязательная ссылка в тексте |

### Требования к формулам (ГОСТ)

| Элемент | Требование |
|---------|------------|
| Нумерация | Справа в круглых скобках |
| Пояснения | Значения символов после формулы |
| Ссылки | Номер формулы в тексте |

### Требования к списку источников (ГОСТ)

| Элемент | Требование |
|---------|------------|
| Количество | ≥15 источников |
| Оформление | Алфавитно-хронологическая последовательность |
| Ссылки | На каждый источник должна быть ссылка в тексте |
| Элементы | Автор, название, издательство, год издания |

### Технические требования к БД (ГОСТ 34.601-90)

| Требование | Значение |
|------------|----------|
| ER-модель | ≥10 сущностей |
| Связи M:N | 1-2 связи «многие ко многим» |
| Нормализация | Приведение к 3НФ |
| СУБД | Любая, кроме Access |
| Триггеры | ≥3 штуки |
| Представления | Для запросов конечного пользователя |
| Формы | Интерфейс для ввода данных |
| Отчёты | Документы, имитирующие внутренние документы фирмы |
| Безопасность | Разграничение прав доступа по ролям |

## Пользовательские требования

> Требования, добавленные для конкретной курсовой работы

### Требования к содержанию

1. **Таблицы в главах** — глава не может содержать только таблицу, должна быть подводка к этой таблице
   - Минимум **4-6 предложения** или **60 слов** в подводке
   - Подводка должна содержать объяснение, анализ или выводы по данным таблицы, а не просто название

2. **Рисунки и диаграммы** — глава не может содержать только рисунок/диаграмму, должна быть подводка к ней
   - Минимум **2-3 предложения** в подводке
   - Подводка должна содержать объяснение, описание или анализ того, что изображено на рисунке
   - После подводки идёт сам рисунок с подписью (например, Рисунок 1.1 — Название)

3. **SQL-код и листинги** — перед кодом должна быть подводка
   - Минимум **1-2 предложения**
   - Подводка объясняет, что делает этот код, для чего он нужен

4. **Mermaid диаграммы** — формат для ER-диаграмм и других схем
   - Использовать формат `erDiagram` для ER-диаграмм
   - Сущности и связи должны быть на русском языке
   - Типы связей: `||--o{` (один-ко-многим), `||--||` (один-к-одному), `}|--|{` (многие-ко-многим)
   - Описание связей на русском в кавычках (например: "ведёт", "включает", "заключает")
   - **Все атрибуты сущностей должны быть описаны в таблице раздела**, диаграмма содержит только связи
   - Диаграмма должна помещаться на формате А4

## Ключевые требования (быстрая справка)

### Обязательные элементы курсовой
1. Титульный лист
2. Содержание (оглавление)
3. Введение
4. Основная часть (3 главы с выводами)
5. Заключение
6. Список источников (≥15)
7. Приложения

### Технические требования
- **Объём:** ≥30 страниц
- **СУБД:** Любая, **КРОМЕ Access**
- **ER-модель:** ≥10 сущностей, 1-2 связи «многие ко многим»
- **Триггеры:** ≥3 штуки
- **Литература:** ≥15 источников с ссылками в тексте
