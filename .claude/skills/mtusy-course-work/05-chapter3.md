2. **Примеры представлений:**

## 3.6. Реализация ограничений и автоматизация

**КРИТИЧЕСКИ ВАЖНО:** Не менее **3 триггеров**

### Что сделать:

1. **Ограничения целостности (CHECK, CONSTRAINT)**
   ```sql
   ALTER TABLE Товар
   ADD CONSTRAINT CK_Цена CHECK (цена > 0);
   ```

2. **Триггеры** (минимум 3)
   - Описание логики
   - SQL-код триггера
   - Когда срабатывает

### Примеры триггеров:

```sql
-- Триггер 1: Автоматическое обновление итоговой суммы заказа
CREATE OR REPLACE FUNCTION обновить_сумму_заказа()
RETURNS TRIGGER AS $$
BEGIN
    UPDATE Заказ
    SET сумма = (
        SELECT COALESCE(SUM(количество * цена), 0)
        FROM Состав_заказа
        WHERE id_заказа = NEW.id_заказа
    )
    WHERE id_заказа = NEW.id_заказа;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_обновить_сумму
AFTER INSERT OR UPDATE ON Состав_заказа
FOR EACH ROW EXECUTE FUNCTION обновить_сумму_заказа();
```

```sql
-- Триггер 2: Проверка остатка на складе
CREATE OR REPLACE FUNCTION проверить_остаток()
RETURNS TRIGGER AS $$
DECLARE
    остаток INTEGER;
BEGIN
    SELECT количество INTO остаток
    FROM Склад
    WHERE id_товара = NEW.id_товара;
    
    IF остаток < NEW.количество THEN
        RAISE EXCEPTION 'Недостаточно товара на складе!';
    END IF;
    
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```
