# Чеклист для проверки курсовой работы

> Используйте этот чеклист перед сдачей курсовой работы

---

## Структура работы

- [ ] **Титульный лист** оформлен по образцу
  - [ ] Название университета
  - [ ] Тема работы
  - [ ] ФИО студента
  - [ ] ФИО научного руководителя
  - [ ] Год

- [ ] **Содержание** присутствует
  - [ ] Все разделы перечислены
  - [ ] Номера страниц указаны
  - [ ] Соответствует структуре работы

- [ ] **Введение** написано
  - [ ] Актуальность темы обоснована
  - [ ] Цель работы сформулирована (глаголы: разработать, спроектировать, создать)
  - [ ] Задачи соответствуют главам (4-6 задач)
  - [ ] Объект и предмет исследования указаны
  - [ ] Методы перечислены
  - [ ] Структура работы описана

- [ ] **Глава 1** написана
  - [ ] Объект автоматизации описан (ООО «Название»)
  - [ ] Организационная структура приведена
  - [ ] Бизнес-процессы описаны
  - [ ] Группы пользователей выделены
  - [ ] Задачи каждой группы описаны
  - [ ] Входные и выходные документы перечислены
  - [ ] Обзор ИТ-технологий проведён
  - [ ] CASE-средства описаны
  - [ ] Обзор аналогов выполнен (3-4 системы)
  - [ ] Требования к БД сформулированы
  - [ ] Выводы по главе написаны

- [ ] **Глава 2** написана
  - [ ] ER-модель разработана
  - [ ] Не менее 10 сущностей
  - [ ] 1-2 связи «многие ко многим»
  - [ ] ER-диаграмма представлена (рисунок)
  - [ ] Выбор модели данных обоснован
  - [ ] Переход к даталогической модели выполнен
  - [ ] Схема БД представлена (рисунок)
  - [ ] Нормализация проведена (до 3НФ)
  - [ ] SQL-код создания таблиц приведён
  - [ ] Выводы по главе написаны

- [ ] **Глава 3** написана
  - [ ] Выбор СУБД обоснован
  - [ ] СУБД **НЕ Access**
  - [ ] База данных создана
  - [ ] Таблицы созданы (SQL-код)
  - [ ] Внешние ключи описаны
  - [ ] Представления (Views) созданы
  - [ ] Кириллица в представлениях
  - [ ] Формы описаны
  - [ ] Скриншоты форм приложены
  - [ ] Отчёты описаны
  - [ ] Отчёты имитируют документы фирмы
  - [ ] **Минимум 3 триггера** реализовано
  - [ ] SQL-код триггеров приложен
  - [ ] Безопасность описана
  - [ ] Роли и права описаны
  - [ ] Выводы по главе написаны

- [ ] **Заключение** написано
  - [ ] Результаты перечислены
  - [ ] Выводы сделаны
  - [ ] Предложения приведены

- [ ] **Список источников** оформлен
  - [ ] **Минимум 15 источников**
  - [ ] Оформлен по ГОСТ
  - [ ] Источники упорядочены

---

## Технические требования

- [ ] **Объём:** не менее 30 страниц
- [ ] Формат A4
- [ ] Поля: левое 30 мм, остальные 15-20 мм
- [ ] Шрифт Times New Roman, 14 пт
- [ ] Межстрочный интервал 1.5
- [ ] Нумерация страниц
- [ ] Рисунки пронумерованы и подписаны
- [ ] Таблицы пронумерованы и подписаны
- [ ] Ссылки на рисунки и таблицы в тексте

---

## Содержательные требования

- [ ] Текст соответствует теме
- [ ] Нет прямого копирования источников (должны быть ссылки)
- [ ] Все источники из списка литературы использованы
- [ ] Выводы соответствуют содержанию глав
- [ ] Нет противоречий между главами
- [ ] Терминология единообразная

---

## SQL и код

- [ ] SQL-запросы синтаксически верны
- [ ] Использована выбранная СУБД (PostgreSQL/MySQL/...)
- [ ] Триггеры имеют описание логики
- [ ] Код в приложениях соответствует тексту

---

## Перед сдачей

- [ ] Проверить орфографию и пунктуацию
- [ ] Проверить нумерацию страниц
- [ ] Проверить оглавление
- [ ] Распечатать и переплести
- [ ] Проверить, что всё на месте
